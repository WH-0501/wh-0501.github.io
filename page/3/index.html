任务

发表于 2024-11-16 分类于 嵌入式 ， FreeRTOS 本文字数： 2.4k 阅读时长 ≈ 4 分钟

FreeRTOS 中，每个执行线程都被称为任务。

前台系统：是单任务系统。单片机裸机一般是在 main 函数中用 while 循环来完成所有的处理，即应用是一个无限循环，循环中调用相应的函数完成相应的处理，有时也会需要在中断中完成一些处理。

前台系统中，中断服务函数称为前台程序，外层的 while 循环为后台程序。前台系统各任务依次轮流执行，实时性差。

FreeRTOS 是一个抢占式实时多任务系统(并不是同一时刻执行多个任务)。FreeRTOS 的任务调度器也是抢占式的。高优先级的任务可以打断低优先级任务的运行而取得 CPU 的使用权，以确保紧急任务的运行。

任务与协程(Co-Routine)

任务和协程使用不同的 API 函数，因此不能通过队列(或信号量)将数据从任务发送给协程，反之亦然。协程是为那些资源很少的 MCU 准备的，其开销很小，但是目前 MCU 性能越来越强大，协程已经很少用到，且 FreeRTOS 官方已经不再更新协程了。

不同点：

① 堆栈。所有协程使用同一个堆栈，而每个任务都有自己的堆栈。因此协程消耗更少的 RAM

② 协程使用合作式调度器，但可以在抢占式的调度器中使用协程

③ 协程通过宏定义实现

④ 为了降低 RAM 消耗，做了很多限制

任务状态

阻塞状态：任务正在等待某个事件。任务可进入阻塞态以等待以下两种不同类型的事件：

  • 定时事件

  • 同步事件

挂起状态：调用 vTaskSuspend() 接口进入挂起状态，并调用 vTaskResume()/vTaskResumeFromISR() 接口唤醒进入就绪状态等待系统调度

任务优先级

每个任务都可以分配一个从 0 ~ configMAX_PRIOROTIES-1 的优先级(configMAX_PRIOROTIES 在 FreeRTOSConfig.h 中定义)。FreeRTOS 本身并没有限定 configMAX_PRIOROTIES 的最大值，但这个值越大，则内核花销的内存空间就越多。所以总是建议将此常量设为能够用到的最小值。

数越小，对应的优先级越低。

configUSE_TIME_SLICING 为 1 时，多个任务可共用一个优先级，此时处于就绪状态的优先级相同的任务会使用时间片轮转调度器获取运行时间。

任务优先级也可在调度器启动后调用 vTaskPrioritySet() 接口修改。

调度器在以时间片为单位来切换任务执行。

时间片的长度通过 tick 中断的频率进行设定，tick 中断频率由 configTICK_RATE_HZ 进行配置。

FreeRTOS API 函数调用中指定的时间总是以 tick 中断为单位。常量portTICK_RATE_MS 用于将以 tick 为单位的时间值转化为以毫秒为单位的时间值。有效精度依赖于系统 tick 频率。

tick count（tick 计数器）表示从调度器启动开始，tick 中断的总数。用户程序在指定延迟周期时不必考虑 tick count 溢出问题。

任务实现

FreeRTOS 中，使用 xTaskCreate() 和 xTaskCreateStatic() 来创建任务。任务函数格式为：

  • 任务函数不应该返回
  • 任务函数的参数为 void 指针类型
  • 任务具体执行过程是一个大循环，循环中为任务功能代码
  • 任务函数一般不允许跳出循环，若要跳出循环，需要调用 vTaskDelete() 函数删除任务

在创建任务时，会自动为每个任务分配一个任务控制块 TCB_t，存储任务属性等内容。

空闲任务

为保证系统至少有一个任务处于运行态，当调用 vTaskStartScheduler() 时，调度器会自动创建一个空闲任务（优先级为 0）。

通过空闲任务钩子函数(或称回调，hook, or call-back)，可以直接在空闲任务中添加应用程序相关的功能。空闲任务钩子函数会被空闲任务每循环一次就自动调用一次。

空闲任务钩子函数通常被用于：

  • 执行低优先级，后台或需要不停处理的功能代码
  • 测试系统处理裕量(空闲任务只会在所有其它任务都不运行时才有机会执行，所以测量出空闲任务占用的处理时间就可以清楚的知道系统有多少富余的处理时间)
  • 将处理器配置到低功耗模式

空闲任务钩子函数的限制：

  • 绝对不能阻塞或挂起。空闲任务只会在其它任务都不运行时才会被执行(除非有应用任务共享空闲任务优先级)。以任何方式阻塞空闲任务都可能导致没有任务能够进入运行态
  • 如果应用程序用到了 vTaskDelete() 函数，则空闲钩子函数必须能够尽快返回。因为在任务被删除后，空闲任务负责回收内核资源。如果空闲任务一直运行在钩子函数中，则无法进行回收工作

任务堆栈

创建任务时需要指定堆栈，若使用 xTaskCreate() 创建任务，则会自动创建堆栈；若使用 xTaskCreateStatic() 创建任务，则需要手动指定任务堆栈。

空闲任务的堆栈大小为：configMINIMAL_STACK_SIZE。

任务同步

FreeRTOS 中可使用信号量、事件标志组来实现同步。

使用信号量进行同步，任务只能与单个的事件或任务进行同步。使用事件标志组可以实现任务与多个事件或任务进行同步。

从 FreeRTOS v8.2 版本开始，新增了任务通知(Task Notifications)功能，可替代信号量、消息队列、事件标志组等。

信号量

FreeRTOS中信号量又分为二值信号量、计数型信号量、互斥信号量和递归信号量。

二值信号量与互斥信号量类似，但互斥信号量拥有优先级继承机制，二值信号量没有优先级继承。

内存管理

发表于 2024-11-16 分类于 嵌入式 ， FreeRTOS 本文字数： 582 阅读时长 ≈ 1 分钟

任务、队列或信号量被创建时，内核需要进行动态分配。

FreeRTOS 调用 pvPortMalloc() 请求内存，调用 vPortFree() 释放内存。

FreeRTOS 有五种内存分配范例，对应：heap_1.c、heap_2.c、heap_3.c、heap_4.c、heap_5.c。

heap_1

实现了基本的 pvPortMalloc()，没有 vPortFree()，即：不支持内存释放。

将 FreeRTOS 的内存堆空间看做一个简单数组，调用 pvPortMalloc 时，将数组简单的细分为更小的内存块。

heap_2

采用最佳匹配算法来分配内存，支持内存释放。

最佳匹配算法保证 pvPortMalloc 会使用最接近请求大小的空闲内存块。

heap_2 不会对相邻的空闲块进行合并，因此会产生内存碎片。

heap_3

简单调用标准库函数 malloc 和 free。此时的内存堆空间不受 configTOTAL_HEAP_SIZE 影响，而由链接器配置决定。

heap_4

heap4 相比于 heap2，可以把内存碎片合并成大块内存。

heap_5

heap_1、2 和 4 方式都是静态申请一个数组作为堆，Heap_5 允许使用多个不连续的区域组成堆，申请前，必须通过函数 vPortDefineHeapRegions() 进行设置，对于一些内存分布不连续的嵌入式设备还是很有价值的，之后其他操作，和 Heap_4 基本一致。

STM32-中断

发表于 2024-11-16 分类于 嵌入式 ， STM32 本文字数： 1.9k 阅读时长 ≈ 3 分钟

常见内部中断：复位、不可屏蔽中断(NMI)、硬件异常等。内部中断不能被打断，不能设置优先级(优先于外部中断)。

中断嵌套即打断其他正在执行的中断。

STM32 有两个中断优先级：抢占式优先级(主)和响应优先级(次)

抢占优先级任何比任何响应优先级高。只有抢占优先级更高的才具有中断嵌套功能；抢占优先级相同的中断，高响应优先级的不可以打断低响应优先级的中断；抢占优先级相同的中断，在同时发生时，响应优先级高的先执行。例如：

假定设置中断优先级组为2，然后设置中断3(RTC_WKUP中断)的抢占优先级为2，响应优先级为1。中断6（外部中断0）的抢占优先级为3，响应优先级为0。中断7（外部中断1）的抢占优先级为2，响应优先级为0。那么这3个中断的优先级顺序为：中断7>中断3>中断6。

Cortex-M3定义了 8bits 用于设置中断优先级。 Cortex-M3 允许具有较少中断源时使用较少的寄存器指定中断源的优先级，因此，STM32中，则把中断优先级寄存器位减少到 4bits。

NVIC(Nested Vector interrupt controller)：中断控制器，负责除了 SYSTICK 之外的所有中断的控制。但 NVIC 只能配置 16 级中断优先级。库中将中断分为 5 组，分别为：

NVIC_PriorityGroup NVIC_IRQChannelPreemptionPriority(抢占优先级) NVIC_IRQChannelSubPriority(响应优先级) Description
NVIC_PriorityGroup_0 0 0-15 0位抢占优先级,4位响应优先级
NVIC_PriorityGroup_1 0-1 0-7 1位抢占优先级,3位响应优先级
NVIC_PriorityGroup_2 0-3 0-3 2位抢占优先级,2位响应优先级
NVIC_PriorityGroup_3 0-7 0-1 3位抢占优先级,1位响应优先级
NVIC_PriorityGroup_4 0-15 0 4位抢占优先级,0位响应优先级

中断优先级分组在一个程序中只能设定一次。每个中断可设置抢占优先级为 0~7(越小优先级越高)，响应优先级位 1 或 0。

STM32 的外部中断通过边沿触发(上升沿触发、下降沿触发和双边沿触发)，不支持电平触发。

STM32 总共有 16 根中断线：EXTI0~EXTI15。

中断相关寄存器：

  • ISER：Interrupt Set-Enable Registers 中断使能寄存器组
  • ICER：Interrupt Clear-Enable Registers 中断清除寄存器组
  • ISPR：Interrupt Set-Pending Registers 中断挂起控制寄存器组
  • ICPR：Interrupt Clear-Pending Registers 中断解除挂起控制寄存器组
  • IABR：Interrupt Active Registers 中断激活寄存器组
  • IP[240]：Interrupt Priority Regitsters 中断优先级寄存器组

STM32 编程步骤：

① 配置时钟，包括中断时钟、系统时钟等

② 设置GPIO(根据需要)

③ EXTI 配置，包括：中断源的选择、清除中断标志、触发模式等

④ NVIC 设置(即:配置中断优先级)，例如：所属组、指定信息等

⑤ 编写中断服务子程序(中断服务函数 EXTIn_IRQHandler 和中断处理回调函数 HAL_GPIO_EXTI_IRQHandler)

默认情况下，调用 HAL_Init 之后，会设置中断优先级分组组号为 4。可修改 HAL 库中的 HAL_Init 接口，修改默认分组组号。例如，STM32F429IGT，找到 stm32f4xx_hal.c：

STM32 中断服务函数注意事项：

① 中断服务函数函数名是固定的，在启动代码中已经确定下来

② 先清中断标志，再执行中断处理(如果把清除中断标志放在中断服务函数的最后，会出现当发出清中断标志指令后，硬件还没有把相关标志清除掉，程序就已经跳出了中断服务函数，这个时候NVIC又会识别到标志是1，出现重复中断)。如果要先执行中断处理再清中断标志，需要注意等待清除成功后再往下执行

③ 中断服务函数应尽量简短，不要在中断中做延时等长时间占用 CPU 的操作

④ 中断服务函数不会被任何函数调用

⑤ 中断服务函数不能返回值

⑥ 中断服务函数不能传递参数

STM32 启动过程指从上电开始，直到运行 main 函数之间的过程。

三种启动模式

  • Main Flash memory：从用户闪存Flash启动(正常工作模式)，一般使用 JTAG/SWD 下载程序都是下载到这里面。起始地址为 0x8000000，复位后 PC 指针位于 0x8000000；

  • Embedded memory：从内置 SRAM 启动。通常用于程序调试。起始地址为 0x2000000，复位后 PC 指针位于 0x2000000

STM32-定时器及PWM

发表于 2024-11-16 分类于 嵌入式 ， STM32 本文字数： 2.1k 阅读时长 ≈ 4 分钟

STM32 共有11个定时器

2个高级控制定时器(TIM1、TIM8)

4个普通定时器(TIM2 - TIM5)

2个基本定时器(TIM6、TIM7)

2个看门狗定时器

1个系统嘀嗒定时器(SysTick)

TIM1 结构

TIM1由一个16位的自动装载计数器组成，由一个可编程预分频器驱动。适合多种用途，包含测量输入信号的脉冲宽度，或者产生输出波形。使用定时器预分频器和RCC时钟控制预分频器，可以实现脉冲宽度和波形周期从几个微秒到几个毫秒的调节。

SysTick

是一个周期性的、倒计时定时器，用于提供时基，多为操作系统使用。SysTick 定时器被捆绑在 NVIC 中，用于产生 SYSTICK 异常(异常号:15)。SysTick 有四个寄存器：

  • SysTick->CTRL ：控制和状态寄存器

  • SysTick->LOAD ：重装载寄存器

  • SysTick->VAL ：当前值寄存器

  • SysTick->CALIB ：校准值寄存器

Systick 的操作步骤：

① 设置 SysTick 时钟源

② 设置 SysTick 的重装初始值

③ 清除 SysTick 当前计数器的值

④ 使能 SysTick 定时器

使用 Systick 实现 delay 时有两种方式：

  • 中断方式

    • a) 定义延时时间 time
    • b) 调用 SysTick_Config() 定义中断时间段，例如: SysTick_Config(SystemFrequency/1000) - 中断间隔 1ms；SysTick_Config(SystemFrequency/100000) - 中断间隔 10μs
    • c) 在 SYSTICK 中断处理函数中递减 time

  • 非中断方式 - 通过 SysTick 四个寄存器实现

主要寄存器

计数寄存器(TIMx_CNT)

预分频寄存器(TIMx_PSC)

自动装载寄存器(TIMx_ARR)

重复次数寄存器(TIMx_RCR)

捕获/比较模式寄存器(TIMx_CCMR1/2)

捕获/比较使能寄存器(TIMx_CCER)

捕获/比较寄存器(TIMx_CCR1~4)

普通定时器时钟来源

  • 内部时钟(CK_INT)

  • 外部时钟模式1：外部输入引脚(TIx)

  • 外部时钟模式2：外部触发输入(ETR)

编程步骤

① 配置系统时钟,例如：RCC_Configuration()

② 配置 NVIC(嵌套中断控制器的配置)

③ 配置 GPIO,例如：GPIO_cfg() (可选,如需要定时操作GPIO等)

④ 配置 TIMER,例如：Timer_Config()

系统时钟(RCC)配置,例如:

① 将RCC寄存器恢复为默认值 RCC_DeInit

② 打开外部高速时钟晶振 RCC_HSEConfig

③ 等待外部高速时钟晶振工作

④ 设置AHB时钟 RCC_HCLKConfig

⑤ 设置高速APB时钟 RCC_PCLK2Config

⑥ 设置低速APB时钟 RCC_PCLK1Config

⑦ 设置PLL锁相环 RCC_PLLConfig

⑧ 打开PLL锁相环 RCC_PLLCmd

⑨ 等待PLL锁相环工作 while(RCC_GetFlagStatus(RCC_FLAG_PLLRDY)==RESET)

⑩ 设置系统时钟 RCC_SYSCLKConfig

⑪ 判断PLL是否为系统时钟 while(RCC_GetSYSCLKSource()!=0x08)

⑫ 打开要使用的外设时钟 RCC_APB2PeriphClockCmd/RCC_APB1PeriphClockCmd

timer配置过程

① 利用TIM_DeInit()函数将Timer设置为默认缺省值

② TIM_InternalClockConfig()选择TIMx来设置内部时钟源；

③ TIM_Perscaler来设置预分频系数；

④ TIM_ClockDivision来设置时钟分割；

⑤ TIM_CounterMode来设置计数器模式；

⑥ TIM_Period来设置自动装入的值；

⑦ TIM_ARRPerloadConfig()来设置是否使用预装载缓冲器；

⑧ TIM_ITConfig()来开启TIMx的中断

计数器模式

向上计数模式、向下计数模式、中央对齐模式

向上计数模式：计数器从0计数到自动加载值(TIMx_ARR计数器的内容)，然后重新从0开始计数并且产生一个计数器溢出事件

向下计数模式：计数器从自动加载值(TIMx_ARR计数器的内容)向下计数到0，然后重新从自动加载值开始计数并且产生一个向下溢出事件

中央对齐模式：计数器从0开始计数到自动加载的值(TIMx_ARR寄存器)-1，产生一个计数器溢出事件，然后向下计数到1并且产生一个计数器下溢事件；然后再从0开始重新计数

PWM

ARR 决定 PWM 周期

CCRx 决定 PWM 占空比（高低电平所占比例）

改变 CCRx 的值，就可以改变 PWM 输出的占空比，改变 ARR 的值，就可以改变 PWM 输出的频率

Fpwm = 时钟频率/((ARR+1)*(PSC+1))

其中：PSC 为预分频系数

例如：假设 TIM3 时钟，即 APB1 Timer Clocks 为 72MHz，假设要求 PWM 周期为 500Hz（即2ms）

设置 PSC(预分频系数)： 720 - 1 = 719，则计数器计数单位为 0.1MHz = 10μs；要求为 500Hz即2ms，则 ARR 应为：199

STM32-WDG

发表于 2024-11-16 分类于 嵌入式 ， STM32 本文字数： 340 阅读时长 ≈ 1 分钟

WDG：看门狗。STM32 有两个看门狗：独立看门狗(IWDG)和窗口看门狗(WWDG)。

IWDG:

  • 使用内部低速时钟 LSI，在主时钟故障时仍能正常工作
  • 无中断
  • 12位递减
  • 只有下限
  • 适合用于主程序之外，能完全独立工作，并且对事件精度要求低的场合
  • 一般用于避免程序跑飞或死循环

WWDG:

  • 使用系统时钟 APB1ENR (由APB1时域分频得到)
  • 有中断
  • 7位递减
  • 有下限和上限
  • 适合用于要求看门狗在精确计时窗口起作用的应用程序
  • 一般用于避免程序不按预定逻辑执行

IWDG 使用步骤：

① 取消寄存器写保护

② 设置 IWDG 预分频系数，确定时钟

③ 设置看门狗重装载值，确定溢出时间

④ 使能看门狗

⑤ 喂狗

WWDG 使用步骤：

① 使能看门狗时钟

② 设置分频值，确定时钟

③ 设置窗口值

④ 开启 WWDG 中断并分组

⑤ 编写中断服务函数

⑥ 使能看门狗

⑦ 喂狗

复用与重映射

复用 - STM32 所有内置外设(例如：I2C、ADC、ISP、USART等)的外部引脚都是与标准 GPIO 引脚复用的，如果有多个复用功能模块对应同一个引脚，只能使能其中之一，其它模块保持非使能状态。

重映射 - 复用功能的引出脚可以通过重映射，从不同的 I/O 管脚引出，即复用功能的引出脚位是可通过程序改变到其他的引脚上

③. I/O 重映射开启

④. 配置重映射引脚，选择需要的功能，去除不需要的

STM32-DMA

发表于 2024-11-16 分类于 嵌入式 ， STM32 本文字数： 1.3k 阅读时长 ≈ 2 分钟

DMA(Direct Memory Access)。DMA传输方式无需 CPU 直接控制传输，也没有中断处理的保留现场和恢复现场的过程，通过硬件为 RAM 与 I/O 设备开辟直接数据传输通路，大大提高 CPU 效率。

DMA 传输过程的五个事件标志：

  • DMA 半传输
  • DMA 传输完成
  • DMA 传输错误
  • DMA FIFO 错误
  • 直接模式错误

DMA控制器执行直接存储器传输：因为采用AHB主总线，它可以控制AHB总线矩阵来启动AHB事务。它可以执行下列事务：

  • 外设到存储器的传输
  • 存储器到外设的传输
  • 存储器到存储器的传输
  • 外设到外设的传输

存储器到存储器需要外设接口可以访问存储器。STM32F429 中，只有 DMA2 的外设接口可以访问存储器。

DMA 主要配置参数：源地址、目的地址、传输数据量。

STM32Cubex 配置 DMA(以 STM32F103C8T6 为例)

参数说明：

  • Mode：

    • Normal：buffer 满了就不再接收
    • Circular：buffer 满了后会覆盖

  • Increment Address：地址自加。DMA 中外设就相当于一个单一的寄存器，地址不自增；而存储器，接收一个数据就存储，然后地址自增，指向下一个写入的地址

  • Data Width：数据宽度

    • Byte
    • Half Word
    • Word

    在32位机中，Word = 32bit，Half word = 16bit，Byte = 8bit；在16位机中，Word = 16bit，Half word = 8bit，Byte = 8bit；在 8 位机中，Word = 16bit，Half word = 8bit，Byte = 8bit

发表于 2024-11-16 分类于 嵌入式 ， STM32 本文字数： 2.5k 阅读时长 ≈ 5 分钟

IAP(In Application Programming)：目的是为了在产品发布后能够通过预留的通信口对产品中的固件程序进行在线升级。

该方法不同于 ICP(In-Circuit Programming) 及 ISP。ICP通过在线仿真器对单片机程序进行烧写；而 ISP 则是通过单片机内置的 bootloader 程序(System memory 中的 bootloader)引导的烧写技术。ICP 和 ISP 均需要连接下载线等操作。

IAP 的前提是必须是基于可重复编程闪存的微控制器。STM32 带有可编程的内置内存。

IAP 的核心是一段预先烧录在单片机内部的 IAP 程序(BootLoader)，该程序主要负责与外部的上位机软件进行同步，然后通信接口将来自上位机软件的程序数据接收后写入单片机内部制定的闪存区域，然后跳转执行新写入的程序。

使用 IAP 过程写入的程序需满足两个要求：

  • 新程序必须从 IAP 程序之后的某个偏移量为 x 的地址开始

IAP 使用注意事项：

  • 应用程序要编译成 bin 格式(hex文化包含了数据和地址信息，bin文件只包含数据)
  • IAP 程序不需要从串口/USB口/其它接口接收数据，固件传输由应用程序自行完成
  • 接收完升级程序数据之后，校验，并复位系统
  • IAP 程序只检查固件升级标志，若未设置固件升级标志，则加载 Flash 中的 APP 程序
  • 若 APP 设置了固件升级标志，固件地址及固件加密方式，IAP 程序会根据这些信息，进入固件升级流程
  • 固件升级流程：解除 Flash 的 lock 状态/读写保护，读取外部存储器中的固件，对固件解密后写入内部 Flash，重新开启读保护和写保护

要点：

  • flash 擦除及写入：flash操作过程：flash解锁、flash擦除、flash写入、flash上锁

    • a) flash 解锁 - FLASH_Unlock()/HAL_FLASH_Unlock()
    • b) flash 擦除 - FLASH_ErasePage()/HAL_FLASHex_Erase()
    • c) flash 写入 - FLASH_ProgramHalfWord()/HAL_FLASH_Program()
    • d) flash 上锁 - FLASH_Lock()/HAL_FLASH_Lock()

  • 中断向量表的偏移设置：对于BootLoader程序只需设置中断向量表的指向在 0x08000000 处，假设用户程序需要设置中断向量表的指向在 0x08008000 处：

    • NVIC_SetVectorTable(NVIC_VectTab_FLASH, 0x0);
    • NVIC_SetVectorTable(NVIC_VectTab_FLASH, 0x8000);
    • 其中: NAVIC_VectTable_FLASH 为 0x08000000
