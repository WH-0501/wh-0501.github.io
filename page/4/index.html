STM32-DMA

发表于 2024-11-16 更新于 2024-12-13 分类于 嵌入式 ， STM32 本文字数： 1.5k 阅读时长 ≈ 3 分钟

DMA(Direct Memory Access)。DMA传输方式无需 CPU 直接控制传输，也没有中断处理的保留现场和恢复现场的过程，通过硬件为 RAM 与 I/O 设备开辟直接数据传输通路，大大提高 CPU 效率。

DMA 传输过程的五个事件标志：

  • DMA 半传输
  • DMA 传输完成
  • DMA 传输错误
  • DMA FIFO 错误
  • 直接模式错误

DMA控制器执行直接存储器传输：因为采用AHB主总线，它可以控制AHB总线矩阵来启动AHB事务。它可以执行下列事务：

  • 外设到存储器的传输
  • 存储器到外设的传输
  • 存储器到存储器的传输
  • 外设到外设的传输

存储器到存储器需要外设接口可以访问存储器。STM32F429 中，只有 DMA2 的外设接口可以访问存储器。

DMA 主要配置参数：源地址、目的地址、传输数据量。

STM32Cubex 配置 DMA(以 STM32F103C8T6 为例)

参数说明：

  • Mode：

    • Normal：buffer 满了就不再接收
    • Circular：buffer 满了后会覆盖

  • Increment Address：地址自加。DMA 中外设就相当于一个单一的寄存器，地址不自增；而存储器，接收一个数据就存储，然后地址自增，指向下一个写入的地址

  • Data Width：数据宽度

    • Byte
    • Half Word
    • Word

    在32位机中，Word = 32bit，Half word = 16bit，Byte = 8bit；在16位机中，Word = 16bit，Half word = 8bit，Byte = 8bit；在 8 位机中，Word = 16bit，Half word = 8bit，Byte = 8bit

串口 + DMA

  • 初始化串口（USART）：设置波特率、字长、停止位、校验位等参数，使能串口

  • 配置 DMA：包括外设基地址、内存基地址、传输方向、数据宽度、传输模式等

  • 使能DMA中断（可选）：如果需要在数据传输完成后进行某些操作，可以配置DMA中断

  • 使能串口 DMA 请求

  • 配置 NVIC：配置中断寄存器，使能 DMA 中断(可选)

STM32STM32-IAP

发表于 2024-11-16 更新于 2024-12-13 分类于 嵌入式 ， STM32 本文字数： 2.8k 阅读时长 ≈ 5 分钟

IAP(In Application Programming)：目的是为了在产品发布后能够通过预留的通信口对产品中的固件程序进行在线升级。

该方法不同于 ICP(In-Circuit Programming) 及 ISP。ICP通过在线仿真器对单片机程序进行烧写；而 ISP 则是通过单片机内置的 bootloader 程序(System memory 中的 bootloader)引导的烧写技术。ICP 和 ISP 均需要连接下载线等操作。

IAP 的前提是必须是基于可重复编程闪存的微控制器。STM32 带有可编程的内置内存。

IAP 的核心是一段预先烧录在单片机内部的 IAP 程序(BootLoader)，该程序主要负责与外部的上位机软件进行同步，然后通信接口将来自上位机软件的程序数据接收后写入单片机内部制定的闪存区域，然后跳转执行新写入的程序。

使用 IAP 过程写入的程序需满足两个要求：

  • 新程序必须从 IAP 程序之后的某个偏移量为 x 的地址开始
  • 必须将新程序的中断向量表相应的移动，移动偏移量为 x。对应接口：void NVIC_SetVectorTable(u32 NVIC_VectTab, u32 Offset)

IAP 使用注意事项

  • 应用程序要编译成 bin 格式(hex文化包含了数据和地址信息，bin文件只包含数据)
  • IAP 程序不需要从串口/USB口/其它接口接收数据，固件传输由应用程序自行完成
  • 接收完升级程序数据之后，校验，并复位系统
  • IAP 程序只检查固件升级标志，若未设置固件升级标志，则加载 Flash 中的 APP 程序
  • 若 APP 设置了固件升级标志，固件地址及固件加密方式，IAP 程序会根据这些信息，进入固件升级流程
  • 固件升级流程：解除 Flash 的 lock 状态/读写保护，读取外部存储器中的固件，对固件解密后写入内部 Flash，重新开启读保护和写保护

要点：

  • flash 擦除及写入：flash操作过程：flash解锁、flash擦除、flash写入、flash上锁

    • a) flash 解锁 - FLASH_Unlock()/HAL_FLASH_Unlock()
    • b) flash 擦除 - FLASH_ErasePage()/HAL_FLASHex_Erase()
    • c) flash 写入 - FLASH_ProgramHalfWord()/HAL_FLASH_Program()
    • d) flash 上锁 - FLASH_Lock()/HAL_FLASH_Lock()

  • 中断向量表的偏移设置：对于BootLoader程序只需设置中断向量表的指向在 0x08000000 处，假设用户程序需要设置中断向量表的指向在 0x08008000 处：

    • NVIC_SetVectorTable(NVIC_VectTab_FLASH, 0x0);
    • NVIC_SetVectorTable(NVIC_VectTab_FLASH, 0x8000);
    • 其中: NAVIC_VectTable_FLASH 为 0x08000000

BootLoader 实现过程

BootLoader 的主要作用是在操作系统内核运行之前，对系统的硬件设备进行初始化、建立内存空间的映射图，将系统的软、硬件设置成一个合适的环境，以便为最终调用操作系统内核做好准备。

BootLoader 两个阶段：

  • Stage 1：
    • 硬件设备初始化：负责初始化最基本的硬件设备，如时钟系统、内存控制器等
    • 为加载 BootLoader 的 Stage2 准备 RAM 空间
    • 复制 BootLoader 的 Stage2 到 RAM 空间中
    • 设置好堆栈
    • 跳转到 Stage2 的 C 入口点
  • Stage 2：
    • 初始化本阶段要使用的硬件设备：进行更详细的硬件初始化，包括外设、中断控制器、DMA等
    • 检测系统内存映像
    • 将 kernel 映像和根文件系统映像从 Flash 读到 RAM 中

UART及USART

发表于 2024-11-16 本文字数： 731 阅读时长 ≈ 1 分钟

USART(Universal Synchronous and Asynchronous Receiver and Transmitter): 通用同步和异步收发器

UART(Universal Asynchronous Receiver and Transmitter): 通用异步收发器

同步：发送数据后，等待应答再发下一个数据。阻塞模式

异步：无需等待应答。非阻塞模式

总线信号：

UART - TX RX

USART - TX RX CK

异步通信时，两者无差别。

USART 多了同步通信功能，因此可以将 USART 当做 SPI 使用。

STM32 串口使用：

  • 使能串口时钟。串口属于外设，其时钟由外设时钟使能寄存器控制
  • 串口配置，包括: 设置 IO 模式、波特率、数据位长度、奇偶校验位、收发模式等信息

串口传输方式：

  • 查询
  • 中断
  • DMA

对应 HAL 库 API:

  • HAL_UART_Transmit

  • HAL_UART_Receive

  • HAL_UART_Transmit_IT // 中断

  • HAL_UART_Receive_IT

  • HAL_UART_Transmit_DMA // DMA

  • HAL_UART_Receive_DMA

串口发送/接收数据：

  • 查询方式效率非常低;

  • 中断方式，如果每个字节触发一个中断，效率也不高，且需要自行解包;

  • 空闲中断可实现按帧传输。空闲中断：串口空闲时产生的中断。若两帧数据之间间隔超过一定时间，接收端就可以产生空闲中断。使用空闲中断：

    • 可以接收不定长数据
    • 不需要复杂帧格式（如：不需要帧头/帧尾）
    • 一帧错误不影响后续帧

  • DMA + 空闲中断的方式可以进一步提升传输效率

AMBA 总线规范(ARM公司提出)。定义了 AHB(Advanced High-performance Bus)、ASB(Advanced System Bus)、APB(Advanced Peripheral Bus)。

AHB：用于高性能、高时钟频率的系统结构，典型的有：内部高速 RAM、Nand Flash、DMA、Bridge等；

APB：用于连接外部设备

AHB 总线系统

特性：

  • burst transfers(突发传输)

  • split transactions()

  • single cycle bus master handover()

  • single clock edge operation

  • non-tristate implementation

  • wider data bus configurations (64/128 bits).

STM32主系统由 32位 多层 AHB 总线矩阵构成。总线矩阵使用 循环调度算法 控制总线之间的访问仲裁管理。可实现如下部分的互联：

  • 八条主控总线是：

    • Cortex-M4内核I总线, D总线和S总线;
      • I 总线(S0)：指令总线。内核通过该总线获取指令，访问的对象是包括代码的存储器
      • D 总线(S1)：数据总线。内核通过该总线进行立即数加载和调试访问
      • S 总线(S2)：系统总线。该总线用于访问外设或 SRAM 中的数据

    • DMA2外设总线;
    • 以太网DMA总线;
    • USB OTG HS DMA总线;

  • 七条被控总线：

    • 内部FLASH ICode总线;

    • 内部FLASH DCode总线;

    • 主要内部SRAM1(112KB)

    • 辅助内部SRAM2(16KB);

    • 辅助内部SRAM3(64KB)(仅适用STM32F42xx和STM32F43xx系列器件);

    • AHB1外设和AHB2外设;

    • FSMC

CubeMX 配置

SYS 配置

Timebase source 配置的是 HAL 的时基。

裸机运行时，需要维护的时间仅有 SYS Timebase Source，也就是HAL库中的 uwTick。主要用于实现 HAL_Delay() 以及作为各种 timeout 的时钟基准。裸机运行时，通常选择 SysTick 即可。

带 OS 运行时，OS 也需要一个时钟基准，来对任务和时间进行管理。该时基一般是 SYSTICK。此时需要考虑 HAL 时基 和 OS 时基是否共用 Systick。（注：CubeMX 生成 FreeRTOS 代码时会提示不要使用 SYSTICK 作为 HAL 时基）

使能 FreeRTOS

STM32CubeMx 使能 FreeRTOS 后，NVIC 配置中 Priority Group 只能选择: 4 bits for pre-emption priority 0 bits for subpriority

FreeRTOS 的中断配置没有处理亚优先级(即:响应优先级)这种情况，因此只能配置为分组 4，即: 16 个抢占优先级。这样中断优先级的管理将非常方便，这也是官方建议的方式。

烧录

四种烧录方式

  • JTAG
    主要引脚：电源、复位、TDI(数据输入)、TDO(数据输出)、TMS(模式选择)、TCK(时钟)
    实现工具：J-Link
  • SWD
    串行调试口，JTAG通常与SWDIO接口复用。只需两根线(不包括VCC、GND)：SWCLK(串行时钟线)、SWDIO(串行数据线)
    实现工具：J-Link
  • ISP
    串口烧录。与单片机连接时，要交叉连接，即：TXD 接 RXD。STM32使用 UART 下载程序时，需要选择 BOOT 的模式，但现在通过 CH340G 芯片(USB转串口)可实现自动 ISP 的下载方式，不再需要手动设置 BOOT 模式(需要先切到 SYSTEM 模式通过串口下载代码，然后再切换回来)。System Memory中有一段 BOOTLOADER 来接收串口来的数据，把它们烧写到FLASH中。
  • SWIM
    实现工具：如 ST-Link
  • IAP
    需要在 Flash 中写入一段操作flash的程序。

STM8 系统通过 SWIM 接口与 ST-Link/V2 连接

STM32 系列通过 JTAG/SWD 接口与 ST-Link/V2 连接

hex 与 flash

hex 大小和需要的 flash 空间没有必然关系。有时 hex 大于 flash 时也能正常烧录，因为实际写入 flash 中的是 hex 中的 bin 文件。

keil 编译结束时会提示如下信息：

  • Code：程序中代码字节数
  • RO-data：常量字节数
  • RW-data：已初始化的变量的字节数
  • ZI-data：未初始化的变量字节数

flsh = Code + RO-data + RW-data

RAM = RW-data + ZI-data

三种启动模式

STM32 系列支持三种启动模式：

  • Main Flash memory：STM32 内置 flash，一般使用 JTAG/SWD 下载程序都是下载到这里面
  • System memory：系统存储器是芯片内部特定区域。通常为 Bootloader，即:ISP。选用此模式时，通常是为了通过串口来下载程序，因为原厂提供的 Bootloader 都提供了串口下载程序的固件。使用 Bootloader 经串口下载程序时，需要注意 BOOT0 及 BOOT1 状态的切换
  • Embedded memory：内置 SRAM。通常用于程序调试

串口下载步骤

  • BOOT0接 V3.3（BOOT1保持接 GND）
  • 按下复位按钮

即可开始下载，下载完成后，若未设置从 0x08000000 开始运行，则代码不会立即执行，此时，还需将 BOOT0 切换回 GND，再按一次复位才会开始运行刚下载的代码。

正点原子 STM32F429 开发板一键下载电路，使用串口的 DTR 和 RTS 信号，分别控制 STM32 的复位和 BOOT0。可通过上层软件 Flymcu ，设置:DTR低电平复位，RTS高电平进 Bootloader。

寄存器配置过程

ST 官方提供 HAL(硬件抽象层API) 库，该库负责与寄存器直接交互，并向用户提供调用 API。

时钟源

STM32 启动过程中，在进入 main 函数之前，会执行 SytemInit 来设置系统时钟。

相关术语：

  • RCC：Reset and Clock Control，复位和时钟控制。用于管理复位和时钟信号的关键模块。

时钟源

STM32 中有五个时钟源：

  • HSI：High Speed Interal 高速内部时钟。由内部RC振荡器产生，频率通常为8/16MHz，精度较高，适用于对时钟精度要求较高的场合
  • HSE：High Speed External 高速外部时钟
  • MSI：Middle Speed Interal 中速内部时钟。频率范围为100kHz~4MHz，精度介于HSI和LSI之间，适用于对时钟精度要求中等的场合
  • LSI：Low Speed Interal 低速内部时钟。频率约为32kHz，精度较低，适用于对时钟精度要求不高的场合，如实时时钟（RTC）等
  • LSE：Low Speed External 低速外部时钟
  • PLL：高速时钟。分为 主PLL、PLLI2S(I2S专用)、PLLSAI(SAI专用)。PLL可以将 HSI 或 HSE 时钟信号进行倍频，产生更高频率的时钟信号，从而提高系统的性能

HSE、LSE是外部时钟，通过外接晶振的方式获取，其它为内部时钟。

SYSCLK

系统时钟 SYSCLK 的三个来源：HSI、HSE、PLL

RTC 时钟源

RTC时钟来源：LSE、LSI、HSE的128分频

IWDGCLK

看门狗时钟 IWDGCLK 来源：LSI

USBCLK

USB 时钟 USBCLK 来源：只能从 PLL 输出端获取。当需要使用USB模块时，PLL必须使能，并且需要配置其时钟频率为 48MHz。如果经过 PLL 倍频后输出 72MHZ，需要再经USB预分频器1.5分频后再供给USB使用

外设时钟源

系统时钟(SYSCLK)通过 AHB 分频器给外设提供时钟。AHB 分频器输出的时钟作为模块的输入：

  • 给 AHB 总线、core、内存和 DMA 使用的 HCLK
  • 系统时钟定时器
  • 直接给 cortex 的空闲运行时钟 FCLK
  • APB1 外设。主要是低速外设，例如:电源接口、can、USB、I2C、USART2、USART3、UART4、SPI2~3等
  • APB2 外设。主要是高速外设，例如:UART1、SPI1、Timer1、ADC1~3、所有普通 I/O 等(具体可通过查看 STM32 参考手册中的 “系统架构/结构” 部分)

注意：当 APBx 预分频系数为 1 时，TIMxCLK = PCLKx。否则，TIMxCLK = 2 * PCLKx。(通过观察在 STM32CubeMX 中改变 APB 预分频系数的结果可以看出)

RCC

Reset and Clock Control，复位和时钟控制。RCC 模块可以选择不同的时钟源，包括 HSI、HSE、LSI 和 LSE 等。

STM32CubeMX 中配置时

此处配置外部时钟源：

  • BYPASS Clock Source：旁路时钟源。系统时钟由外部提供
  • Crystal/Ceramic Resonator：晶体/陶瓷振荡器。系统时钟由外部晶振和芯片内部电路协作提供

MCO

Microcontroller Clock Output（即：STM32CubeMX RCC 配置中的 Master Clock Output），微控制器时钟输出。允许将内部时钟信号输出到特定的引脚上，通常这个引脚是 PA8。通过 MCO，可以将系统时钟、外部时钟或内部 RC 振荡器的时钟信号输出到外部，供外部设备使用或进行时钟信号的测试和调试

MCO配置步骤：

  • 设置 PA8 为复用推挽输出模式：首先需要配置 GPIOA 的第 8 个引脚（PA8）为复用推挽输出模式，以便于输出时钟信号
  • 选择输出时钟源：通过RCC的CFGR寄存器中的MCO[2:0]位来控制输出的时钟源
  • 配置MCO的分频系数：可以选择不同的分频系数来控制输出时钟的频率，例如，如果外部振荡器是25MHz，可以选择不进行分频（RCC_MCO1Div_1），或者选择分频以降低输出频率

注意：STM32 的 GPIO 输出管脚的最大响应频率为 50MHz，如果输出频率超过 50MHz，则输出的波形会失真。

HAL 时钟系统配置

系统启动之后，会执行 HAL 库定义的 SystemInit 函数，进行系统的一些初始化配置：

  • FPU 设置
  • 复位 RCC 时钟配置为默认复位值(默认开始了 HSI)
  • 外部存储器配置
  • 中断向量表地址配置

HAL 库中的 SystemInit 并没有像标准库中的 SystemInit 一样进行时钟初始化配置，除了打开 HSI 之外，没有任何时钟相关配置。因此，使用 HAL 库时，必须自己配置时钟。

Stm32_Clock_Init 函数的作用是进行时钟系统配置，除了配置 PLL 相关参数确定 SYSCLK 之外，还配置了 AHB、APB1、APB2 的分频系数，即:确定了 HCLK、PCLK1、PCLK2 的时钟值。

使用 HAL 库配置 STM32 时钟的一般步骤;

  • 使能 PWR 时钟 __HAL_RCC_PWR_CLK_ENABLE()。因为后面的设置调节器输出电压级别和开启 over-drive 功能都是电源相关配置，所以必须先开启 PWR 时钟
  • 设置调压器输出电压级别 __HAL_PWR_VOLTAGESCALING_CONFIG()
  • 选择是否开启 Over-Driver 功能 HAL_PWREx_EnableOverDrive()
  • 配置时钟源相关参数 HAL_RCC_OscConfig()
  • 配置系统时钟源及 AHB、APB1、APB2 的分频系数 HAL_RCC_ClockConfig()

设置好时钟系统之后，若还使用到了外设，如:GPIO、ADC等，则还需要使能这些外设时钟。

IO复用

IO引脚会通过一个复用器连接到内置外设或模块。该复用器采用 16 路复用功能输入，可通过 GPIOx_AFRL 和 GPIOx_AFRH 寄存器进行配置。每 4 为控制一路复用。

SysTick

SysTick 定时器计数一次为: 1/180μs，因此 1μs 要计数 180 次。

若 SysTick 时钟设置为 SYSTICK_CLKSOURCE_HCLK_DIV8(即: 180/8 MHz)，则 SysTick 定时器计数一次为: 8/180μs，因此 1μs 要计数 180/8 次。

DMA 发送注意

通过调用 HAL_UART_Transmit_DMA 即可实现利用 DMA 通过 UART 发送数据 tx_data。该函数发送完一帧之后，接着发送下一帧，而不管 tx_data 是否更新。假设上一次发送的 0x0001，当前要发送 0xFFFE，当 tx_data 更新了 0xFF，尚未更新后面的 0xFE(即：上一次的 0x01 尚未被更新)，就被 DMA 发送出去，此时 DMA 发送的是 0xFF01。
