ping

  • -n count：定义用来测试所发出测试包个数，缺省值为 4。例如：向测试发送 20 个数据包的平均返回时间是多少，最快时间是多少，最慢时间是多少，就可使用该选项
  • -l length：发送缓冲区大小。定义发送数据包大小，默认情况下 windows ping 发送的数据包为 32Bytes，使用该参数可自定义大小，但最大只能 65500Bytes
  • -f：不分段标志。一般数据包会分为几段发送，使用该选项后就不会分段发送了
  • -i ttl：指定 TTL 只在对方系统中停留市场
  • -v tos：将服务类型(type-of-server)设置为 tos 指定的值
  • -r count：在记录路由字段中记录传出和返回的数据包的路由。通过该选项可设定想探测经过的路由个数。即：记录跃点数的路由
  • -s count：计数跃点的时间戳 ms
  • -4/-6：强制使用 IPv4/6

ARP 表

查看 neighbour/arp cache

最后一个字段显示该条目的“邻居不可达检测”机器的状态：

  • STALE – The neighbour is valid, but is probably already unreachable, so the kernel will try to check it at the first transmission.
  • DELAY – A packet has been sent to the stale neighbour and the kernel is waiting for confirmation.
  • REACHABLE – The neighbour is valid and apparently reachable.

添加 ARP

其中，neighbour state(nud)：

  • permanent：永久有效，只能通过管理方式删除
  • noarp：邻居条目有效。不会尝试验证此条目，但可以在其生命周期到期时将其删除
  • stale：邻居条目有效但可疑。如果 ip neigh 的此选项有效且该命令未更改地址，则该选项不会更改邻居状态
  • reachable：邻居条目在可达性超时到期之前一直有效

ipcalc

指令安装： sudo apt install ipcalc

  • -c/–check: 检查 IP 地址是否为非法地址
  • -4/–ipv4: 验证 IPv4 地址
  • -6/–ipv6: 验证 IPv6

  • -h/–hostname: 显示给定IP地址所对应的主机名
  • -m/–netmask: 由给定的IP地址计算器网络掩码
  • -p/–prefix: 显示给定的掩码或IP地址的前缀
  • -n/–network: 由给定的IP地址和网络掩码计算网络地址
  • -s/–silent: 安静模式

curl

curl 是常用的命令行工具，用来请求 Web 服务器。

nc

功能强大的网络工具，扫描端口。其作用包括：

  • 实现任意TCP/UDP端口的侦听，nc可以作为server以TCP或UDP方式侦听指定端口
  • 端口的扫描，nc可以作为client发起TCP或UDP连接
  • 机器之间传输文件
  • 机器之间网络测速

nslookup

nslookup - name server lookup。查询 DNS 的记录，查询域名解析是否正常。

指令格式： nslookup domain [dns-server]

两种模式：

  • 交互：命令行直接输入 nslookup 回车
  • 非交互：指令后跟要查询的域名/IP地址

dig

  • -x：反向查询。即：已知 DNS，查询该 DNS 域名
  • @<DNS>：从指定的 DNS 服务器上查询
  • +trace：跟踪整个查询过程

常见 DNS 记录的类型

类型 作用
A 地址记录，用来指定域名的 IPv4 地址，如果需要将域名指向一个 IP 地址，就需要添加 A 记录
AAAA 用来指定主机名(或域名)对应的 IPv6 地址记录
CNAME 如果需要将域名指向另一个域名，再由另一个域名提供 ip 地址，就需要添加 CNAME 记录
MX 如果需要设置邮箱，让邮箱能够收到邮件，需要添加 MX 记录
NS 域名服务器记录，如果需要把子域名交给其他 DNS 服务器解析，就需要添加 NS 记录
SOA 这种记录是所有区域性文件中的强制性记录。它必须是一个文件中的第一个记录
TXT 可以写任何东西，长度限制为 255。绝大多数的 TXT记录是用来做 SPF 记录(反垃圾邮件)

traceroute&tracepath

traceroute 和 tracepath 均用于 确定数据包从源主机到目标主机所采取的路径。

两者的区别：

  • 工作原理：Traceroute 在数据包中加入一个 TTL（Time to Live）值，每次经过一个路由器时此值减一。当 TTL 的值减为零时，路由器应该丢弃数据包，然后向源主机发送一条 ICMP echo 消息。通过检测这些回显消息，Traceroute 工具就可以确定从源主机到目标主机所经过的所有路由器；Tracepath 使用 MTU 探测技术，自动检测通过每个路由器的最大传输单位（MTU），以便有效地确定路径
  • 使用方式：Traceroute 使用命令 traceroute 或 tracert，默认使用 ICMP 数据包，并且 需要管理员权限；Tracepath 使用命令 tracepath，它默认使用 UDP 数据包，并且不需要管理员权限
  • 结果显示：Traceroute 显示 每个路由器的 IP 地址、域名、RTT（Round Trip Time）和 TTL 值；Tracepath显示 每个路由器的 IP 地址、域名、RTT（Round Trip Time）和 MTU 值
  • 功能：Traceroute 可以使用 -n 选项来阻止 DNS 解析，以加快操作速度；Tracepath 可以使用 -p 选项来指定要使用的端口号

显示 *，可能防火墙封掉了 ICMP 的返回信息；或者添加 -I 选项。其它解决方法：

  • 配置 ICMP 超时报文功能：ip ttl-expires enable
  • 关闭 tracert 报文共计防范功能：undo firewall defend tracert enable 谨慎使用

traceroute

参数：

  • -d: 使用Socket层级的排错功能
  • -f: 设置第一个检测数据包的存活数值TTL的大小
  • -F: 设置勿离断位
  • -g: 设置来源路由网关，最多可设置8个
  • -i: 使用指定的网络界面送出数据包
  • -I: 使用ICMP回应取代UDP资料信息
  • -m: 设置检测数据包的最大存活数值TTL的大小
  • -n: 直接使用IP地址而非主机名称
  • -p: 设置UDP传输协议的通信端口
  • -r: 忽略普通的Routing Table，直接将数据包送到远端主机上
  • -s: 设置本地主机送出数据包的IP地址
  • -t: 设置检测数据包的TOS数值
  • -v: 详细显示指令的执行过程
  • -w: 设置等待远端主机回报的时间
  • -x: 开启或关闭数据包的正确性检验

注意，每次检测都同时发送 3 个数据包，因此打印出来三个时间。此外，如果某一个数据报超时没有返回，则时间显示为 *

mtr

mtr 来源于 Matt’s TraceRoute。是一个网络诊断工具，将 ping 和 traceroute 命令的功能合二为一。例如： mtr -n www.baidu.com

结果中的 * 表示：可能是防火墙封掉了ICMP的返回信息，或者未在预期的超时时间内确认数据包。

netstat

netstat的输出结果可以分为两个部分:

  • Active Internet connections 有源TCP连接，其中”Recv-Q”和”Send-Q”指接收队列和发送队列。这些数字一般都应该是0。如果不是则表示软件包正在队列中堆积。这种情况只能在非常少的情况见到。
  • Active UNIX domain sockets 有源Unix域套接口(和网络套接字一样，但是只能用于本机通信，性能可以提高一倍)。

列名解释：

  • Proto：显示连接使用的协议。
  • RefCnt：表示连接到本套接口上的进程号。
  • Types：显示套接口的类型。
  • State：显示套接口当前的状态。
  • Path：表示连接到套接口的其它进程使用的路径名。

常用选项：

  • -t : 指明显示TCP端口，t 是 TCP 的首字母
  • -u : 指明显示UDP端口，u 是 UDP 的首字母
  • -l : 仅显示监听套接字，l 是 listening 的首字母
  • -p : 显示进程标识符和程序名称，每一个套接字/端口都属于一个程序，p是program的首字母
  • -n : 不进行DNS轮询，显示IP(可以加速操作)，n是numeric的首字母，以数字形式显示地址和端口号。否则显示主机名等信息
  • -a : 显示所有连接和侦听端口，a是all的首字母
  • -o : 显示拥有的每个连接关联的进程id，o是own的首字母
  • -c : 持续输出
  • -x : Unix 套接字

ss 查看 socket 使用

ss 指令：(或 cat /proc/net/sockstat)

  • -a, –all: 显示监听和未监听(即:已创建的连接)的socket
  • -l, –listening只显示监听的套接字(这些套接字默认省略)
  • -m, –memory查看socket使用的内存
  • -n, –numeric不解析服务名称
  • -r, –resolve尝试解析数字地址/端口
  • -p, –processes显示socket对应的进程
  • -i, –info显示内部 TCP 信息
  • -s, –summary打印统计摘要. This option does not parse socket lists obtaining summary from various sources. It is useful when amount of sockets is so huge that parsing /proc/net/tcp is painful.
  • -o, –options显示 timer 信息
  • -e, –extended显示详细信息
  • -t, –tcp只显示 TCP 套接字
  • -u, –udp只显示 UDP 套接字
  • -d, –dccp显示 DCCP 套接字
  • -w, –raw显示 RAW 套接字
  • -x, –unix显示 Unix domain 套接字
  • -f FAMILY, –family=FAMILY显示指定协议族(例如: AF_INET 等)套接字
