为 cmake 设置快捷方式

基本语法

变量使用 ${} 方式取值，但是在 IF 控制语句中是直接使用变量名。

指令 ( 参数 1 参数 2 … ) 参数使用括弧括起，参数之间使用空格或分号分开。 以 ADD_EXECUTABLE 指令为例：ADD_EXECUTABLE(hello main.c func.c) 或者 ADD_EXECUTABLE(hello main.c;func.c) 。

指令是大小写无关的，参数和变量是大小写相关的。但，推荐全部使用大写指令。

cmake 语法灵活，例如： SET(SRC_LIST main.c) 也可写成 SET(SRC_LIST "main.c") ，但是当文件名中有空格时则必须使用双引号；还可以忽略源文件列表中的文件后缀， cmake 会自动在本目录查找。

当工程中有多个子目录时，每个子目录都需创建 CMakeLists.txt 文件

内部构建与外部构建

内部构建即在项目 CMakeLists.txt 同级目录进行配置及编译。外部构建即在项目外进行编译，例如：在项目中创建 <project>/build 目录，然后进入该目录执行 cmake .. && make 配置及配置。

外部构建时，<project>_SOURCE_DIRS 仍为项目根目录，而 <project>_BINRAY_DIR 则为 <project>/build

基础指令

project

定义项目名称， 必要，多个 projcet 指令以最后一个为准。当从顶层调用 CMakeLists.txt 时，cmake 还将项目名称存储在变量 CMAKE_PROJECT_NAME 中。

同时设置以下变量：

  • PROJECT_SOURCE_DIR 、**<PROJECT-NAME>_SOURCE_DIR** ：项目源目录的绝对路径
  • PROJECT_BINARY_DIR 、**<PROJECT-NAME>_BINARY_DIR** ：项目可执行文件目录的绝对路径
  • PROJECT_IS_TOP_LEVEL 、**<PROJECT-NAME>_IS_TOP_LEVEL** ：v3.21 新增变量，表示当前项目是否为 top-level

ADD_LIBRARY

生成动态/静态库。

对于 SHARED 和 MODULE，会自动将 POSITION_INDEPENDENT_CODE 设置为 ON。

若一个库不暴露任何 symbols，则不能被定义为 SHARED 库。

ADD_SUBDIRECTORY

该指令用于向当前工程添加存放源文件的子目录，并可以指定中间二进制和目标二进制存 放的位置。

EXCLUDE_FROM_ALL 参数的含义是将这个目录从编译过程中排除，比如，工程 的 example ，可能就需要工程构建完成后，再进入 example 目录单独进行构建

V3.25 新增 SYSTEM 参数，子目录的 SYSTEM directory 属性将被设置为true。该属性用于初始化在该子目录中创建的每个非导入目标的SYSTEM属性。

DESTINATION 定义了安装的路径，如果路径以 / 开头，那么指的是绝对路径，这时候 CMAKE_INSTALL_PREFIX 就无效了。如果希望使用 CMAKE_INSTALL_PREFIX 来定义安装路径，就要写成相对路径，即不要以 / 开头，那么安装后的路径就是 ${CMAKE_INSTALL_PREFIX}/

FILES 与 programs

两者指令使用方法相同，不同的是安装后的权限。

FILES 默认权限为：OWNER_WRITE, OWNER_READ, GROUP_READ, 和 WORLD_READ ，即 644

programs 权限为：WNER_EXECUTE, GROUP_EXECUTE, 和 WORLD_EXECUTE，即 755

DIRECTORY

目录安装。注意: abc 表示将被安装到目标路径下的 abc ,而 abc/ 表示将 abc 中的内容安装到目标路径，不包括该目录本身 。

参数 PATTERN 用于使用正则表达式进行过滤， PERMISSIONS 用于指定 PATTERN 过滤后的文件 权限。

不论是 SUBDIRS 还是 ADD_SUBDIRECTORY 指令 ( 不论是否指定编译输出目录 ) ，都可 以通过 SET 指令重新定义 EXECUTABLE_OUTPUT_PATH 和 LIBRARY_OUTPUT_PATH 变量 来指定最终的目标二进制的位置 ( 指最终生成的 hello 或者最终的共享库，不包含编译生成 的中间文件 )

常用变量

  • <project>_BINARY_DIR 工程根目录

  • PROJECT_NAME 返回通过PROJECT指令定义的项目名称

  • CMAKE_BINARY_DIR 、**PROJECT_BINARY_DIR** 运行cmake命令的目录,通常是 ${PROJECT_SOURCE_DIR}/build

  • PROJECT_SOURCE_DIR 工程的根目录

  • CMAKE_SOURCE_DIR

  • <project>_SOURCE_DIR

  • CMAKE_INCLUDE_PATH 环境变量,非cmake变量

  • CMAKE_LIBRARY_PATH 环境变量

  • CMAKE_CURRENT_SOURCE_DIR 当前处理的CMakeLists.txt所在的路径

  • CMAKE_CURRENT_BINARY_DIR target编译目录。使用 **ADD_SURDIRECTORY(src bin)可以更改此变量的值。SET(EXECUTABLE_OUTPUT_PATH <新路径>)**并不会对此变量有影响,只是改变了最终目标文件的存储路径

  • CMAKE_CURRENT_LIST_FILE 输出调用这个变量的CMakeLists.txt的完整路径

  • CMAKE_CURRENT_LIST_LINE 输出这个变量所在的行

  • CMAKE_MODULE_PATH 定义自己的cmake模块所在的路径。**SET(CMAKE_MODULE_PATH ${PROJECT_SOURCE_DIR}/cmake)**,然后可以用 INCLUDE 命令来调用自己的模块

  • EXECUTABLE_OUTPUT_PATH 重新定义目标二进制可执行文件的存放位置

  • LIBRARY_OUTPUT_PATH 重新定义目标链接库文件的存放位置

  • CMAKE_ALLOW_LOOSE_LOOP_CONSTRUCTS 用来控制 IF ELSE 语句的书写方式

系统信息

  • CMAKE_MAJOR_VERSION cmake主版本号,如2.8.6中的2

  • CMAKE_MINOR_VERSION cmake次版本号,如2.8.6中的8

  • CMAKE_PATCH_VERSION cmake补丁等级,如2.8.6中的6

  • CMAKE_SYSTEM 系统名称,例如Linux-2.6.22

  • CAMKE_SYSTEM_NAME 不包含版本的系统名,如Linux

  • CMAKE_SYSTEM_VERSION 系统版本,如2.6.22

  • CMAKE_SYSTEM_PROCESSOR 处理器名称,如i686

  • UNIX 在所有的类UNIX平台为TRUE,包括OS X和cygwin

  • WIN32 在所有的win32平台为TRUE,包括cygwin

开关选项

  • BUILD_SHARED_LIBS 控制默认的库编译方式。如果未进行设置，使用 ADD_LIBRARY 时又没有指定库类型,默认编译生成的库都是静态库

  • CMAKE_C_FLAGS 设置C编译选项

  • CMAKE_CXX_FLAGS 设置C++编译选项

  • CMAKE_C_COMPILER_INIT

  • **CMAKE_BUILD_TYPE**：Debug、Release、RelWithDebInfo、MinSizeRel。对应使用如下宏来生成 Makefile:

  • CMAKE_SIZEOF_VOID_P: void* 的大小，可用于判断当前系统是 32bits 还是 64bits

cmake自定义变量

主要有隐式定义和显式定义两种.

隐式定义：就是 PROJECT 指令,会隐式的定义_BINARY_DIR 和_SOURCE_DIR 两个变量。

显式定义：使用 SET 指令,就可以构建一个自定义变量了。
