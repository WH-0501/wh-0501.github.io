Docker NVIDIA

使⽤ NVIDIA 显卡，有 nvidia-docker、nvidia-docker2、nvidia-container-toolkits 选项

  • nvidia-docker 是 NVIDIA 第⼀代⽀持 docker 容器内使⽤ GPU 的项⽬，需配套使⽤ nvidia-docker 命令

  • nvidia-docker2 是 NVIDIA 第⼆代⽀持 docker 容器使⽤ GPU 资源的项⽬，仍需配套使⽤ nvidia-docker 命令，并需要指定参数 --runtime=nvidia

  • nvidia-container-toolkits：最新的⽅案。nvidia-docker2 已被 nvidia-container-toolkits 取代，使⽤ --gpus 参数，⽆需再指定 --runtime

nvidia-container-toolkits 包含一个 container runtime library（libnvidia-container） 和用于自动配置容器以利用NVIDIA gpu 的工具。

参考：https://developer.nvidia.com/zh-cn/blog/gpu-containers-runtime/

安装 nvidia-container-toolkits

指定 --gpus all 参数，容器中程序如 RViz 等⽆法正常使⽤ NVIDIA GPU。解决⽅法，容器内部执⾏指令时，指定环境变量

jetson-containers

在Jetson上不推荐直接使用公共 Docker Image，NVIDIA为一些常用的镜像编译了适合Jetson的版本

更改 runtime 为 nvidia，这样 NVCC 编译器和 GPU 才可用

拉取镜像，参考：https://hub.docker.com/r/dustynv/ros/tags。注意，L4T 版本需高于镜像中版本(可通过 jtop 查看)

注：L4T - Linux for Tegra

jetson-container 默认带 --rm 参数，因此 exit 退出时会自动删除容器。

Jetson 中，需确保 docker 镜像中的 OS 版本与 Jetson 环境一致。

例如：JetPack6.0 需使用 dustynv/ros:xxx-l4t-r36.2.0 镜像

在JetPack 5上，CUDA/cuDNN/TensorRT 等被安装在容器内(不像 JetPack 4，它们通过NVIDIA runtime 从主机设备安装到容器中)。

Jetson AGX Orin 32GB，容器镜像 ros:noetic-pytorch-l4t-r34.1.1 ，容器内 CUDA 版本为 11.4。需将容器内 CUDA 升级到 11.8。

方法：手动提取容器内的cuda-compat deb，然后安装cuda-toolkit或cuda-libraries包(只有cuda和cuda-runtime依赖于cuda-compat/nvidia-l4t-core)。

参考：https://github.com/dusty-nv/jetson-containers/issues/258
