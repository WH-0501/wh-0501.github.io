二、测试
先下载测试数据：
2D bag包；3D bag包

cartographer有自己的地图保存方法：
rosservice call /finish_trajectory “map”
(默认会保存在.ros文件夹中)

roslaunch cartographer_ros demo_backpack_2d.launch bag_filename:=2D_bag_file

roslaunch cartographer_ros demo_backpack_3d.launch bag_filename:=3D_bag_file

3D建图过程中显示的是 submap_lists，建图完成后会生成并在 /map主题发布 map 数据

下载基于PR2平台的示例包

wget -P ~/Downloads https://storage.googleapis.com/cartographer-public-data/bags/pr2/2011-09-15-08-32-46.bag

roslaunch cartographer_ros demo_pr2.launch bag_filename:=${HOME}/Downloads/2011-09-15-08-32-46.bag

下载基于Revo LDS平台的示例包

wget -P ~/Downloads https://storage.googleapis.com/cartographer-public-data/bags/revo_lds/cartographer_paper_revo_lds.bag

roslaunch cartographer_ros demo_revo_lds.launch bag_filename:=${HOME}/Downloads/cartographer_paper_revo_lds.bag

三、注意事项
1、发布的occupancy地图只支持2D
3D建图时，实时发布submap_list，建图完成后生成并在/map主题发布2D的occupancy地图数据。rviz中展示的是3D submap数据投影到2D平面的效果，我们将这称之为：x-ray。
cartographer中并没有编写代码用于在cartographer的submaps表示中进行导航或推理、实现轨迹规划和避障。在短期内，可以通过保存轨迹以及创建地图（使用 rosservice call /finish_trajectory），并使用map_server来加载该地图来实现导航。

cartographer目前的形式，相比未知环境下的自主导航，更适合用于建图。因为目前还没有路径规划算法可以根据cartographer的submap来进行规划。
