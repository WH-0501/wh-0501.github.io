编译指令

编译为release版本

CMAKE_BUILD_TYPE 可以的取值是:

  • Debug - 调试信息多，没优化
  • Release - 没有多余的调试信息，发布时使用
  • RelWithDebInfo - 优化过的带有有调试信息的版本
  • MinSizeRel

编译指定包

白名单：**-DCATKIN_WHITELIST_PACKAGES**

恢复编译所有的包（如不执行该指令，则后续都只会编译上述指定的包）

黑名单: -DCATKIN_BLACKLIST_PACKAGES

将指定包编译为release版本

可使用多个 –pkg 参数指定多个包

自动初始化工作空间

编译rovio包。如果工作空间未初始化，可以自动完成工作空间初始化 。

catkin_init_workspace

将当前目录初始化为一个 ROS 工作空间（通常不需要运行该指令，除非想将某个目录指定为 ROS 工作空间）

会在当前目录生成一个 CMakeLists.txt 文件，该文件为 /opt/ros/kinetic/share/catkin/cmake/toplevel.cmake 的软链接。

catkin_make CMakeLists.txt 常用变量

install 相关变量：

- CATKIN_PACKAGE_BIN_DESTINATION

- CATKIN_PACKAGE_ETC_DESTINATION

- CATKIN_PACKAGE_INCLUDE_DESTINATION

- CATKIN_PACKAGE_LIB_DESTINATION

- CATKIN_PACKAGE_PYTHON_DESTINATION

- CATKIN_PACKAGE_SHARE_DESTINATION: 被设置为 ${CATKIN_GLOBAL_SHARE_DESTINATION}/${PROJECT_NAME}

- CATKIN_GLOBAL_BIN_DESTINATION：通常值用于ROS核心执行程序

- CATKIN_GLOBAL_ETC_DESTINATION

- CATKIN_GLOBAL_INCLUDE_DESTINATION

- CATKIN_GLOBAL_LIB_DESTINATION

- CATKIN_GLOBAL_LIBEXEC_DESTINATION

- CATKIN_GLOBAL_PYTHON_DESTINATION

- CATKIN_GLOBAL_SHARE_DESTINATION

catkin_make_isolated

当一个工作空间中包含不同环境的独立的项目时，可以使用该指令进行编译。为纯粹由 catkin 包构成的工作区提供调试，并允许用户构建包含 catkin 和 **非catkin **包的工作区。

非catkin包：编译脚本中不使用 Catkin cmake，而是简单的 CMake 或者其它的工具的程序包。

catkin build

主要区别在于 catkin build 获取的 isolated 环境。catkin build 使得整个 build configuration 更加独立，并且对配置文件的修改(添加/删除程序包，修改 cmake 变量等)更加健壮。

  • catkin build 并行编译相互独立的包，因此速度更快
  • catkin build 可在 workspace 的任何目录下执行，**catkin_make 只能在顶层目录下执行**
  • 编译指定包更加方便。catkin build ，或者，在要编译的包目录下执行 catkin build --this
  • catkin build 输出的日志更易读
  • 使用 catkin build，还可以获得许多其它有用的 catkin 子命令，例如:
    • catkin clean - 清除 build devel 及 install 目录
    • catkin list
    • catkin locate
    • catkin profile
