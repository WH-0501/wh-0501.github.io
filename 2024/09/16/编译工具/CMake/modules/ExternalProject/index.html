外部项目定义

目录选项

大部分情况下，默认目录布局就足够了。主要是通常不需要更改主项目的实施细节。但是，在某些情况下，控制目录布局可能是有用的或必要的。

PREFIX 外部项目的根目录
TMP_DIR 存储零时文件的目录
STAMP_DIR 存储每一步的 timestamps.除非被 LOG_DIR，否则还会在此创建每个步骤的日志文件
LOG_DIR 存储每一步的日志文件的目录
DOWNLOAD_DIR 存放下载的文件(解压前).该参数只有在使用URL下载时有效，其它的下载方法直接使用 SOURCE_DIR
SOURCE_DIR 下载的源码目录
BINARY_DIR 编译目录位置。若指定了 BUILD_IN_SOURCE,则该参数失效
INSTALL_DIR 安装路径

下载选项

如果使用 SOURCE_DIR 指定了本地的非空目录，则会忽略指定的下载方法。否则必须指定下载方法(不能指定多个下载方法)。下载方法包括指定下载指令(DOWNLOAD_COMMAND)、URL下载、Git下载、SVN下载、Mercurial下载、CVS下载等：

DOWNLOAD_COMMAND 覆盖下载步骤的指令(支持 generator expressions)。若指定了该选项，则会忽略所有其它下载选项。将该选项设置为空，可以有效的禁用下载步骤

URL 下载

URL <url1> [<url2>......] 外部项目源的路径。指定多个URL时，会依次尝试直到成功。URL可以是本地文件系统中的普通路径(此时URL必须是唯一的)或file命令支持的任何可下载URL。使用本地文件系统路径时可以引用现有的目录或压缩文件。使用压缩文件时，除非设置DOWNLOAD_NO_EXTRACT选项防止解压，否则将自动解压。压缩格式通过检查实际内容而不是通过扩展名来确定
URL_HASH <algo>=<hashvalue> algo为hashing算法名，例如:MD5等。

update/patch 步骤选项

每当重新运行CMake时，如果下载方法支持更新，则默认情况下将更新外部项目的源

UPDATE_COMMAND …
UPDATE_DISCONNECTED
PATCH_COMMAND …

configure 步骤选项

configure 步骤在下载和更新步骤之后。默认情况下，假设外部项目也是一个 cmake 项目，但可根据需要进行配置

Build 步骤选项

根据 configure 步骤确定默认的编译指令为 cmake 还是普通的 Makefile

BUILD_COMMAND … 重载默认的编译指令。将该选项设置为空字符串，可以使编译步骤不做任何处理
BUILD_IN_SOURCE 在外部项目源码目录进行编译。
BUILD_ALWAYS 强制每次都编译
BUILD_BYPRODUCTS … 指定将由build指令生成的文件，但是其修改时间可能会或可能不会由后续的构建更新。

Install 步骤选项

INSTALL_COMMAND …

Test 步骤选项

TEST_COMMAND …
TEST_BEFORE_INSTALL 在 install 之前执行测试步骤
TEST_AFTER_INSTALL
TEST_EXCLUDE_FROM_MAIN

输出日志选项

LOG_DOWNLOAD 使能下载日志
LOG_UPDATE 使能更新日志
LOG_PATCH 使能Patch日志
LOG_CONFIGURE 使能configure日志
LOG_BULD 使能编译日志
LOG_INSTALL 使能安装日志
LOG_TEST 使能测试日志
LOG_MERGED_STDOUTERR 合并stdout和stderr
LOG_OUTPUT_ONFAILURE 该选项只有在上面的选项中至少使能一项时有效。若设置为 True，则各步骤的错误日志输出到对应文件的同时，也会输出到终端显示。

终端访问选项

在某些情况下，各个步骤可以直接访问终端。

USES_TERMINAL_DOWNLOAD
USES_TERMINAL_UPDATE
USES_TERMINAL_CONFIGURE
USES_TERMINAL_BUILD
USES_TERMINAL_INSTALL
USES_TERMINAL_TEST

Target 选项

DEPENDS … 指定该外部项目依赖的一起目标
EXCLUDE_FROM_ALL 将该外部项目排除在 ALL 目标之外
STEP_TARGETS … 为指定的步骤生成自定义的目标
INDEPENDENT_STEP_TARGETS … 为指定的步骤生成自定义目标，并防止这些目标具有通常的依赖项

Miscellaneous 选项

LIST_SEPARATOR 将所有 …_COMMAND 选项的 ‘;’ 替换为
COMMAND …
