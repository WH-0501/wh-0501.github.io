Docker-compose

发表于 2024-09-22 分类于 Docker 本文字数： 5.2k 阅读时长 ≈ 9 分钟

compose、machine 和 swarm 是 docker 原生提供的三大编排工具。

Compose

Compose 是一个使用 Docker 定义和运行复杂应用程序的工具。使用 Compose，可以在单个文件中定义一个多容器应用程序，然后在单个命令中启动应用程序，该命令会执行运行所需的所有操作。所以基本上，这将有助于缩短需要在 Docker CLI 中键入的内容。

配置文件

Docker Compose 默认配置文件名为 docker_compose.yaml。也可以使用 -f 参数指定具体文件。

Docker Compose 的 YAML 包含 4 个一级 key：

  • version：必须指定，且总是位于文件第一行。指定 Compose 文件格式的版本（注意：不是 Compose 或 docker engine 的版本）。 Compose 文件格式有3个版本,分别为1, 2.x 和 3.x。目前主流的为 3.x 其支持 docker 1.13.0 及其以上的版本。参考：compose-version
  • services：用于定义不同的应用服务。Docker Compose 会将每个服务部署在各自的容器中。services 下面的第一级别的 key 即为 service 的名称
  • networks：定义网络信息。默认情况下，Docker Compose 会创建 bridge 网络。这是一种单主机网络，只能够实现同一主机上容器的连接。当然，也可以使用 driver 属性来指定不同的网络类型
  • volumes：定义卷信息

其它 key：

  • build：配置构建时，Compose 会利用它自动构建镜像，该值可以是一个路径，也可以是一个对象，用于指定 Dockerfile 参数

    • context：指定 Dockerfile 文件路径
    • dockerfile：指定 context 目录下的 Dockerfile 名称
    • args：该 Dockerfile 在 build 过程中需要的参数。(等同于 docker container build –build-arg)
    • cache_from：v3.2 新增参数。指定缓存镜像列表。 (等同于 docker container build –cache_from)
    • labels：v3.3 新增参数。设置镜像元数据。(等同于 docker container build –labels)
    • shm_size：v3.5 新增。设置容器 /dev/shm 分区大小(等同于 docker container build –shm-size)
  • build: ./dir

  • entrypoint：覆盖容器的默认 entrypoint 指令(等同于 docker run –entrypoint)
  • dns：配置 dns 服务器。可以是一个值或列表
  • dns_search：配置 DNS 搜索域。可以是一个值或列表

  • environment：环境变量配置。可以用数组或字典两种方式
  • env_file：从文件中获取环境变量，可以指定一个文件路径或路径列表，其优先级低于 environment 指定的环境变量
  • expose：暴露端口，只将端口暴露给连接的服务，而不暴露给主机
  • image：指定服务所使用的镜像
  • network_mode：设置网络模式

  • ports：对外暴露的端口定义，和 expose 对应

  • links：将指定容器连接到当前连接，可以设置别名，避免ip方式导致的容器重启动态改变的无法连接情况

  • logs：日志输出信息

  • deploy：v3.x 版本，指定与部署和运行服务相关的配置。deploy 部分是 docker stack 使用的, docker stack 依赖 docker swarm

    • endpoint_mode：v3.3 新增，指定服务暴露方式

      • vip：Docker 为该服务分配一个虚拟 IP(VIP)，作为客户端的访问服务的地址

      • dnsrr：DNS 轮询，Docker 为该服务设置 DNS 条目，使得服务名称的 DNS 查询返回一个 IP 地址列表，客户端直接访问其中的一个地址

      • labels：指定服务的标签

      • mode：指定 delopy 的模式

        • global：每个集群节点都只有一个容器
        • replicated：用户可指定集群中容器的数量

      • placement：

      • replicate：mode 为 replicated 时，指定容器副本数量

      • resources：资源限制

        • limits：设置容器资源限制

          • cpus：”0.5” 表示最多只能使用 50% CPU
          • memory：50M 表示最多智能使用 50M 内存空间

        • reservations：设置为容器预留的系统资源

          • cpus：”0.2” 保留 20% 的 CPU
          • memory：20M 保留 20M 内存

      • restart_policy：定义容器重启策略, 用于代替 restart 参数

        • condition：定义容器重启策略(接受三个参数)

          • none：不尝试重启
          • on-failure：只有当容器内部应用程序出现问题才会重启
          • any：无论如何都会尝试重启(默认)

        • delay：尝试重启的间隔时间(默认为 0s)

        • max_attempts：尝试重启次数(默认一直尝试重启)

        • window：检查重启是否成功之前的等待时间(即如果容器启动了, 隔多少秒之后去检测容器是否正常, 默认 0s)

      • update_config：用于配置更新配置

        • parallelism：一次性更新的容器数量

        • delay：更新一组容器之间的间隔时间

        • failure_action：定义更新失败的策略

          • continue：继续更新
          • rollback：回滚更新
          • pause：暂停更新(默认)

        • monitor：每次更新后的持续时间以监视更新是否失败(单位: ns|us|ms|s|m|h) (默认为0)

        • max_failure_ratio：回滚期间容忍的失败率(默认值为0)

        • order：v3.4 版本中新增的参数, 回滚期间的操作顺序

          • stop-first：旧任务在启动新任务之前停止(默认)
          • start-first：首先启动新任务, 并且正在运行的任务暂时重叠

      • rollback_config：v3.7 版本中新增的参数, 用于定义在 update_config 更新失败的回滚策略

        • parallelism：一次回滚的容器数, 如果设置为0, 则所有容器同时回滚

        • delay：每个组回滚之间的时间间隔(默认为0)

        • failure_action：定义回滚失败的策略

          • continue：继续回滚
          • pause：暂停回滚

        • monitor：每次回滚任务后的持续时间以监视失败(单位: ns|us|ms|s|m|h) (默认为0)

        • max_failure_ratio：回滚期间容忍的失败率(默认值0)

        • order：回滚期间的操作顺序

          • stop-first：旧任务在启动新任务之前停止(默认)
          • start-first：首先启动新任务, 并且正在运行的任务暂时重叠

  • devices：指定设备映射列表(等同于 docker run –device)

  • depends_on：定义容器启动顺序

锚点操作符 <<

该描述符用于替代和引用 YAML 中已定义的块。即：将已定义好的模板(可以定义在文件顶部，也可以单独定义)插入到多个不同 service 中，减少荣誉和代码重复。

包含三个符号：

  • &：定义锚点，供以后使用
  • *：引用锚点
  • <<：用于将引用的块内容插入当前位置

--privileged

该选项允许容器以 root 权限运行，获得几乎与主机相同的权限。意味着容器可以访问和控制主机的所有硬件设备、操作系统内核和其他系统资源。

使用 --privileged=true 后，容器内的 root 才真正拥有 root 权限，否则容器内 root 只是一个普通用户

使能该选项存在巨大安全风险。允许容器以超级用户权限运行，意味着容器内的任何恶意代码都可以获得主机上的完全访问权限。攻击者可以利用这种权限执行任意操作，包括但不限于数据窃取、系统破坏和网络]攻击。

-v/volume 及 -device 无法解决热插拔问题。在容器已经运行时插入或重新插入设备，在这种场景中，设备可能获得不同的总线 ID，从而使原始共享失效。（需要注意的是，当共享整个 /dev 目录时，这不是问题）

**内旋(Intrinsic rotation)**：其每个 elemental 绕的是 object space basis 的轴

**外旋(Extrinsic rotation)**：其每个 elemental rotation 绕的是world space basis 的轴

欧拉角旋转序列(Euler Angle Rotational Sequence)共有 12 种顺序：

  • 6 种绕三个轴的旋转，即 Tait-Bryan Angle：

    • XYZ、XZY、YXZ、YZX、ZXY、ZYX

  • 6 种只绕两个轴的旋转，即 Proper Euler Angle：

    • XYX、YXY、XZX、ZXZ、YZY、ZYZ

如果相邻两次旋转是绕同一轴，如 XXY，则可以坍缩为 XY。也就是说，一共有 12 种基础旋转的组合顺序，它们可以旋转出三维的所有旋转状态。

欧拉角与四元数

使用四元数是因为没有万向节死锁问题。

万向节锁是指物体的两个旋转轴指向同一个方向。实际上,当两个旋转轴平行时，即万向节锁现象发生了，换句话说，绕一个轴旋转可能会覆盖住另一个轴的旋转，从而失去一维自由度。

通常说来，万向节锁发生在使用 Eular Angles（欧拉角）的旋转操作中,原因是 Eular Angles 按照一定的顺序依次独立地绕轴旋转。想象一个具体的旋转场景，首先物体先绕转 X 轴旋转，然后再绕 Y 轴，最后绕 Z 轴旋转，从而完成一个旋转操作（实际是想绕某一个轴旋转,然而 Eular Angle 将这个旋转分成三个独立的步骤进行）,当你绕 Y 轴旋转 90 度之后万向节锁的问题就出现了，因为 X 轴已经被求值了，它不再随同其他两个轴旋转，这样 X 轴与 Z 轴就指向同一个方向（它们相当于同一个轴了）。

内参-外参

发表于 2024-09-21 更新于 2024-09-22 本文字数： 929 阅读时长 ≈ 2 分钟

机器人系统中，每个传感器都有各自的坐标系，各传感器得到的测量数据都是基于其自身坐标系。
传感器标定：确定各传感器之间的坐标关系，即相对位姿。

传感器标定可分为：外参标定和内参标定。 内参决定传感器内部映射关系，例如摄像头焦距、畸变系数等，是传感器自身特性，通常由厂家提供，但有部分需要自行标定，例如：相机焦距的定期矫正、lidar中各激光管的垂直朝向角。 外参决定传感器之间的相对位姿，通常由6个自由度表示（旋转+平移）。

不同传感器之间标定的问题在于，每个传感器获取的数据类型不一样：

  • 摄像头得到的是 RGB 图像，但缺乏深度信息
  • 激光雷达得到的是点云，但信号稀疏，噪音大
  • GPS/IMU得到的是机器人的位姿
    因此，不同传感器之间的标定方法不同。

传感器之间的相对位姿关系可通过转换矩阵 M(4X4) 表示。传感器的位移、旋转、尺寸变化都可以用转换矩阵 M 来表示：

包含 3x3 的旋转矩阵 R 和 3x1 的平移矩阵 T。(只需 4x3?)
旋转矩阵 R 具有正交性，且行列值为 1，即： det(R) = 1。 因此，旋转局局长可以转换为轻易装换为 四元素向量。

标定步骤：

  • 粗略测量传感器之间的相对位姿
  • 采集有用的传感器标定数据
  • 算法优化使得标定误差最小化，得到标定值

标定方法分为：targetless 和 target 两种。 前者在自然环境中进行，约束条件少，不需要用专门的target；后者则需要专门的控制场，有ground truth的target，比如典型的棋盘格平面板。

MCL（Monte Carlo Localization）算法

MC(Monte Carlo)方法：
蒙特卡罗方法，也称随机抽样或统计模拟方法。 若所求解的问题是某事件出现的概率或者是某个随机变量的期望值时 。通过某种“实验”的方法，得出事件出现的频率，以此估计出A事件出现的概率，或者得到这个随机变量的期望值。

MCL算法基于粒子滤波器（particle filters）。
Particle filters的基本步骤为：
1、随机生成一堆粒子
粒子可以具有位置，方向或需要估计的任何其他状态变量。 每个都有一个权重（概率），表示它与系统的实际状态匹配的可能性。 用相同的权重初始化每个粒子。
若初始位姿已知，则只需设定粒子群大小 M，然后用repmat方法复制 M 个初始位置作为初始粒子群；若初始位置未知，则可以使用高斯分布随机采样的方法初始化粒子群。
2、预测粒子的下一个状态
根据预测真实系统的行为方式移动粒子。例如根据里程计来数值（速度、加速度）来预测粒子的下一个状态。
3、更新粒子权重
根据测量值更新粒子权重。 每个粒子的加权基于它与测量值的匹配程度。
4、重采样
丢弃极不可能的粒子（权重低的粒子），并用可能性更高的粒子（权重高的粒子）的副本替换它们。
重采样之后，粒子权值归一(权重一致)，但是在机器人可能出现的位置处，粒子数目开始变多。

经过数次迭代后，只有少数粒子权重较大，其余粒子的权值可以忽略不计，粒子权值的方差随着时间增大，空间中的有效粒子数目减少，即：权值退化问题。
5、计算 estimate（可选）
计算该组粒子的均值和协方差以获得最可能的当前状态。

KLD_Sampling_MCL

KLD_Sampling_MCL随时间改变粒子数，改良了度过初期后的蒙特卡洛大样本集合的资源浪费。kld_err 及 kld_z 参数就是KLD的参数。对于每次粒子滤波迭代，KLD采样以概率 1-δ 确定样本数（1-δ就是kld_z配置参数），使得真实的后验与基于采样的近似之间的误差小于ε（ε就是kld_err配置参数）.kld_z=0.99,kld_err=0.05,直方图位大小为15cm15cm15°就能取得良好的结果。

Augmented_MCL

Augmented_MCL解决的是从机器人绑架或全局定位失效中恢复的问题。recovery_alpha_slow 及 recovery_alpha_fast 配置参数就是用于失效恢复的。随机采样以max{0.0,1.0-w(fast)/w(slow)}概率增加。如果短期似然劣于长期似然则增加随机采样，这种方法，测量似然的一个突然衰减将引起随机采样的数目增加。w=w+α(Wavg-w)–Wavg当前测量模型的权重，w为短期（w(fast)）或长期(w(slow))平滑估计，α为与w对应的recovery_alpha_参数。

AMCL参数说明

ROS AMCL节点有三类参数：overall filter、laser model、odometry model。

1、overall filter 参数

min_particles(default: 100)、max_particles(default: 5000)
• 允许的最小粒子数、最大粒子数.

kld_err (double, default: 0.01)
• 实际分布与估算的分布之间的误差的最大值.
kld_z (double, default: 0.99)
• Upper standard normal quantile for (1 - p), where p is the probability that the error on the estimated distrubition will be less than kld_err.
这两个参数是针对 KDL_Sampling_MCL(AMCL的一部分)。

update_min_d (default: 0.2 meters)、update_min_a (default: π/6.0 radians)
• 执行过滤器更新前的移动距离、转动角度。即移动距离或转动角度超过该值就会触发过滤器刷新

resample_interval (int, default: 2)
• 重采样间隔

transform_tolerance (double, default: 0.1 seconds)
• 发布的 map->odom 的 tf 的容错公差.

recovery_alpha_slow (double, default: 0.0 (disabled))
• 慢速平均加权滤波器(slow average weight filter)的指数衰减速率，通过增加随机的位姿来决定何时recover。a good value might be 0.001

recovery_alpha_fast (double, default: 0.0 (disabled))
• 快速平均加权滤波器(fast average weight filter)的指数衰减速率，通过增加随机的位姿来决定何时recover。a good value might be 0.1

initial_pose_x、initial_pose_y、initial_pose_a (default:0,0,0)
• 初始位姿(x,y,yaw), 用于初始化带高斯分布的滤波器(即：初始粒子云分布, initial_pose_x 及 initial_pose_y 确定粒子云的中心位置，initial_pose_a 确定粒子云中粒子的朝向)

initial_cov_xx、initial_cov_yy、initial_cov_aa (default: 0.50.5,0.50.5,0.50.5 meters)
• 初始位姿协方差(xx, yy, yawyaw), 用于初始化带高斯分布的滤波器(即：初始粒子云分布，initial_cov_xx 及 initial_cov_yy 参数决定初始粒子集的范围, initial_cov_aa 确定粒子朝向的偏差)

gui_publish_rate (default: -1.0 Hz)
• scans及paths的最大发布速率(用于可视化)。-1则禁用(即:不发布)

save_pose_rate (double, default: 0.5 Hz)
• 存储最后估算的位姿和协方差(对应于initial_pose_* 及 initial_cov_变量)到参数服务器的速率(即:更新参数服务器中initial_pose_ 及 initial_cov_*的值)。该存储的位姿可用于初始化后续运行的滤波器。-1.0则禁用

use_map_topic (bool, default: false)
• 设置为true时，AMCL将订阅map主题而不是执行服务调用(static_map 服务)来接收地图。

first_map_only (bool, default: false)
• 设置为true时，AMCL将只使用其订阅到的第一张地图，而不会每收到新的地图就进行更新。

2、laser model 参数

注意：使用的混合权重总和始终为1。 beam model 使用所有4个参数： z_hit, z_short, z_max, 及 z_rand。 likelihood_field model 只使用 z_hit 和 z_rand。 通过 laser_model_type 来选择使用的 model。

laser_min_range、laser_max_range (default: -1.0, -1.0)
• 激光有效扫描范围。 激光最小扫描范围。-1.0表示：使用测量到的最小/最大距离，否则，超出该范围的不考虑

laser_max_beams (int, default: 30)
• 更新滤波器时，使用的最大激光束数目.

laser_z_hit(0.95)、laser_z_short(0.1)、laser_z_max(0.05)、laser_z_rand(0.05)
• 分别为模型的 z_hit、z_short、z_max、z_rand 部分的混合权重.

laser_sigma_hit (double, default: 0.2 meters)
• 在模型的z_hit部分中使用的高斯模型的标准偏差.

laser_lambda_short (double, default: 0.1)
• 模型的z_short部分的指数衰减参数.

laser_likelihood_max_dist (double, default: 2.0 meters)
• 在地图上进行障碍物膨胀的最大距离，用于 likelihood_field 模型.

laser_model_type (string, default: “likelihood_field”)
• 选择使用的模型，可以是：beam, likelihood_field, 或者 likelihood_field_prob (与likelihood_field 类似，但合并了 beamskip feature).

3、odometry model 参数

若odom_model_type设置为”diff“，则使用probabilisitic Robotics中p136页中的sample_motion_model_odometry算法，该模式使用噪声参数odom_alpha_1到odom_alpha_4。

若odom_mode_type设置为”omni“，则会为全向底盘使用custom model，该model使用odom_alpha_1到odom_alpha_5参数。前四个参数与”diff” model相似。第五个参数odom_alpha_5采集机器人垂直于观察到的平移方向移动的趋势。
注意：为了修复bug,增加了”diff-corrected”及”omni-corrected”模型。odom_alpha 参数的默认值不再适用于新的 model，新的 model 这些值更低。

odom_model_type(default:”diff”)
• 指定使用的odom模型。可以为：”diff”, “omni”, “diff-corrected” or “omni-corrected”。

odom_alpha1 (0.2)
• 指定在根据机器人的运动的旋转分量估算odometry的旋转时期望的噪声
odom_alpha2 (0.2)
• 指定在根据机器人的运动的平移分量估算odometry的旋转时期望的噪声
odom_alpha3 (0.2)
• 指定在根据机器人的运动的平移分量估算odometry的平移时期望的噪声
odom_alpha4 (0.2)
• 指定在根据机器人的运动的旋转分量估算odometry的平移时期望的噪声
odom_alpha5 (0.2)
• 与平移相关的噪声参数。只对 omni 模型有效

odom_frame_id (default:”odom”)
base_frame_id (default:”base_link”)
global_frame_id (default:”map”)

tf_broadcast(default:true)
• 将此设置为false可禁止发布 全局坐标系 -> odometry坐标系(即: map->odom) 之间的转换。。
