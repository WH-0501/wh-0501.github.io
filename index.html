STM32-ADC

发表于 2024-12-13 分类于 嵌入式 ， STM32 本文字数： 2.2k 阅读时长 ≈ 4 分钟

ADC

12bits ADC 是一种逐次逼近型模数转换器，每个 ADC 有多达 18 个通道，可测量 16 个外部信号源和 2 个内部信号源。

ADC 的输入时钟不得超过 14MHZ，它是由 PCLK2 经分频产生。在 ADCCLK=14M，采样周期为 1.5 个 ADC 时钟。此时的转换时间为 1us

输入电压范围：03.3V，转换结果范围：04095，ADC供电要求：2.4V到3.6V

STM32 ADC的总转换时间为：TCONV = 采样时间 + 12.5个ADC周期。例如， ADCCLK 为 14MHz 时，采样时间为 1.5 个 ADC 周期，此时 TCONV = 14 个 ADC 周期，即 1us。

可设置的采样时间：

  • 1.5 个 ADC 时钟周期
  • 7.5 个 ADC 时钟周期
  • 13.5 个 ADC 时钟周期
  • 28.5 个 ADC 时钟周
  • 41.5 个 ADC 时钟周期
  • 55.5 个 ADC 时钟周期
  • 71.5 个 ADC 时钟周期
  • 239.5 个 ADC 时钟周期

ADC 采集过程

  • 采样
  • 量化：将模拟信号的电压值与ADC的参考电压进行比较，将其转换为相应的数字值
  • 编码：将量化后的结果编码为二进制形式，输出给后续的数字电路或处理器

ADC 采集模式

  • 单次转换模式（Single Conversion Mode）：ADC在每次触发时只进行一次转换，适用于低速、低功耗的应用场景
  • 连续转换模式（Continuous Conversion Mode）：适用于需要实时监测的应用，如信号处理和实时数据采集
  • 扫描模式（Scan Mode）：在多个通道间进行扫描，每个通道依次进行转换，适合多通道数据采集
  • 触发模式（Triggered Mode）：转换过程由外部信号触发，可以是定时器、GPIO引脚等，适合需要同步数据采集的场景
  • 差分模式（Differential Mode）：测量两个输入信号的差值，提供更高的噪声抗性，适用于高精度测量
  • 伪差分模式（Pseudo-Differential Mode）：其中一个输入端连接到地，另一端测量信号，适合简单的差分测量
  • 注入转换模式（Injected Conversion Mode）：用于优先级更高的ADC转换任务，通常与常规转换并行工作
  • 双ADC模式（Dual ADC Mode）：多个ADC模块协同工作，提高采样速度和数据精度

通道选择

有16个通道，可以把转换组织分成两组：

  • 规则组（Regular Channel Group）：由多达16个转换组成，规则通道和它们的转换顺序在ADC_SQRx寄存器中选择，规则组中转换的总数应写入ADC_SQR1寄存器的了L[3:0]位中
    • 常用于周期性采集的传感器信号
    • 优先级较低
  • 注入组（Injected Channel Group）：注入通道组用于优先级更高的信号采集，通常用于突发事件或特定条件下的快速采样。由多达4个转换组成，注入通道和它们的转换顺序在ADC_JSQR寄存器中选择，注入组里的转换总数目应写入ADC_JSQR寄存器的L[1:0]中
    • 优先级较高
    • 通常用于需在特定条件下迅速采集信号的应用，如运动控制、脉冲信号采集等等

编码过程

  • 初始化 ADC：ADC_Init
    • 时钟配置
    • 输入通道选择
    • （可选）启用/禁用 DMA void ADC_DMACmd(ADC_TypeDef *ADCx, FunctionalState NewState)
    • （可选）中断配置 void ADC_ITConfig(ADC_TypeDef *ADCx, uint32_t ADC_IT, FunctionalState NewState)
  • 启动转换：
    • 启用/禁用指定 ADC 外设 void ADC_Cmd(ADC_TypeDef *ADCx, FunctionalState NewState)
    • 开始 ADC 转换 void ADC_StartConversion(ADC_TypeDef *ADCx)
  • 读取结果 uint16_t ADC_GetConversionValue(ADC_TypeDef *ADCx)

相关问题

噪声干扰

优化方法：

  • 添加退耦电容：在ADC的引脚添加一个电容，用于消除噪声。这是因为电容可以吸收电路中的高频噪声，减少干扰。但需注意，电容值不宜过大，以免影响采样的充放电速度，从而影响采样精度
  • 使用 RC 滤波器：在输入信号前添加低通滤波器（如RC滤波），去除高频率噪声，从而减少ADC采样时的噪声干扰
  • 延长采样时间：通过降低ADC时钟频率和延长采样时间，可以减少采样时的噪声。这样可以确保在波形的相同位置进行采样，减少由于噪声引起的误差
  • 软件滤波算法：如一阶互补滤波、中值滤波、算术平均滤波、限幅平均滤波和卡尔曼滤波等，以减少噪声的影响
  • 避免通信外设引起的噪声：在即将开始ADC转换时，避免启动任何通信外设的数据传输，因为I/O的切换可能会在供电电压中产生噪声
  • 避免高灌电流I/O切换：避免切换高灌电流I/O，因为会在电源中产生噪声波纹
  • 避免数字输出在同一I/O端口切换：避免在同一I/O端口上切换数字输出，因为这会将切换噪声引入模拟输入中

串扰问题

通道间串扰问题通常是因为相邻通道之间透过采样电容 Cs 发生了耦合。

解决方法是增大 ADC 相邻通道采样之间的延时：ADC_TwoSamplingDelay

采样时间与外部输入阻抗不匹配

可能导致转换精度降低。

解决办法是根据实际输入阻抗和具体应用来配置合适的采样时间。

TEE：Trusted Exception Environment，是一个安全执行环境，它包括在安全世界中运行的小程序、库和操作系统，用于保护敏感操作和数据

REE：Rich Exception Environment，非安全世界

安全背景

CPU通过内存映射手段给每个进程营造一个单独的地址空间来隔离多个进程的代码和数据，通过内核空间和用户空间不同的特权级来隔离操作系统和用户进程的代码和数据。但由于内存中的代码和数据都是明文，容易被同处于内存中的其它应用偷窥，因此出现了扩展的安全模块，应用将加密数据送往安全模块，由安全模块处理完后再返回结果给相应的应用。

在 TrustZone 之前的安全手段：

  • 外挂硬件安全模块：
    • 数据的处理交由外部的安全模块实现，这些模块能够保护自己的资源和密钥等数据的安全，如SIM卡、各种智能卡或连接到外部的硬件加解密模块等，但其同主芯片的通信线路暴露在外部，容易被监听破解。另外，通信的速率比较低
  • 内部集成硬件安全模块：
    • 片上两个核，一个普通核和一个安全核。安全核性能有限，占用 SoC 面积，成本高

ARM 安全架构

(来源：https://blog.csdn.net/m0_61869253/article/details/139880557)

TrustZone

ARM 公司提出的一种硬件技术，旨在提供处理器级别的安全隔离解决方案，实现对系统资源的安全保护和隔离。通过 TrustZone 技术，处理器可划分为两个独立的区域：

  • Secure World：安全世界
  • Normal World：普通世界

所有需要保密的操作在安全世界执行（如指纹识别、密码处理、数据加解密、安全认证等），其余操作在非安全世界执行（如用户操作系统、各种应用程序等），安全世界和非安全世界通过一个名为Monitor Mode的模式进行转换。

处理器架构上，TrustZone 将每个物理核虚拟为两个核，一个非安全核（Non-secure Core, NS Core），运行非安全世界的代码；和另一个安全核（Secure Core），运行安全世界的代码。两个虚拟的核以基于时间片的方式运行，根据需要实时占用物理核，并通过 Monitor Mode 在安全世界和非安全世界之间切换。

Security states

在 Arm 架构中，有两种安全状态：安全和非安全。

在EL0、EL1和EL2，处理器可以处于安全状态或非安全状态，这是由 SCR_EL3.NS 控制的。表示方法例如：

  • NS.EL1：Non-secure state，Exception level 1
  • S.EL1：Secure state，Exception level 1

不管 SCR_EL3.NS 位的值是多少，EL3 始终在 Secure state。

执行流程：

  • 进入更高的 Exception 级别需要一个异常(exception)。通常，此异常将是 FIQ 或 SMC（Secure Monitor Call - 安全监视调用）异常。即：Rich OS 通过 FIQ/SMC异常进入 EL3

系统架构

一个基于 TrustZone 系统组成

不同芯片厂商会根据安全需求选择不同的安全组件来构造其 TrustZone。其中主要组件包括：

  • 必选组件
    • AMBA3 AXI 总线，安全机制的基础设施
    • 虚拟化的 ARM Core，虚拟安全和非安全核
    • TZPC(TrustZone Protextion Controller)，根据需要控制外设的安全特性
    • TZASC(TrustZone Address Space Controller)，对内存进行安全和非安全区域划分和保护
  • 可选组件
    • TZMA (TrustZone Memory Adapter)，片上ROM或RAM安全区域和非安全区域的划分和保护
    • AXI-to-APB bridge，桥接APB总线，配合TZPC使APB总线外设支持TrustZone安全特性

AMBA3 AXI(AMBA3 Advanced eXtensble Interface)系统总线作为TrustZone的基础架构设施，提供了安全世界和非安全世界的隔离机制，确保非安全核只能访问非安全世界的系统资源，而安全核能访问所有资源，因此安全世界的资源不会被非安全世界（或普通世界）所访问。

总线主设备

A系列处理器都是TrustZone aware的，每次总线访问发送争取的安全状态。但还存在一些非处理器的总线主控，比如GPU、DMA等。

因此将总线主控分为两种：

  • TrustZone aware：每次总线访问都能提供合适安全信息。
  • Non-TrustZone aware：此类主设备访问总显示无法提供安全信息、或每次都是同样安全信息

对于Non-Trusted-aware需要采取的措施有：

  • 设计时固定地址：给设备固定的访问地址
  • 可配逻辑单元：为主设备访问添加安全信息逻辑单元
  • MMU：对于一个可信主设备，SMMU可以向安全状态下的MMU一样处理

软件架构

Top-level software architecture

典型的基于 TrustZone 系统的软件调用栈关系图：

安全世界的Trusted OS提供一系列安全服务，比如key管理或DRM。非安全世界应用需要使用这些服务，但是又无法直接使用。通过使用一些库中API来获取这些能力，比如libDRM.so。

这些库和Trusted service之间通信往往通过message queue或者Mailbox。他们之间通信所用内存往往被称为WSM(World Shared Memory)。这些内存必须能够被Trusted service和库访问，这就意味着这些内存是非安全内存。

应用通过库发送一个请求到Mailbox或message queue，然后出发内核中TrustZone驱动。

TrustZone驱动负责和TEE部分交互，包括为message queue申请内存和注册这些内存。由于安全和非安全运行在两个虚拟地址空间，所以无法通过虚拟地址进行通信。

TrustZone驱动通过调用SMC进入安全状态，控制权通过EL3的Secure Monitor传递到TEE中的TOS。TOS从message queue内存中获取内容个Trusted service进行处理。

  • Trusting the message：由于message是从非安全世界传递的，所以需要安全世界需要对这些内容进行一些认证。
  • Scheduling：对于PSCI类型快速处理并且不频繁请求，进入EL3处理完后退出到非安全状态。对于一些需要TOS处理的任务，不能被非安全中断打断，避免造成安全服务不可用。
  • OP-TEE：OP-TEE内核运行在S.EL1，可信应用运行在S.EL0。

非安全世界的App一般不直接使用TEE Client API，而是中间Service API提供一类服务。TEE Client API和内核中OP-TEE驱动交互，OP-TEE驱动SMC处理底层和OP-TEE内核的通信。

EL3 Firmware/Secure Monitor做SMC处理，根据需要将安全请求发送到OP-TEE内核处理。

MQTT

MQTT（Message Queuing Telemetry Transport，消息队列遥测传输协议），是一种基于 发布/订阅（publish/subscribe） 模式的“轻量级”通讯协议，该协议构建于TCP/IP协议上

除标准版外，还有一个简化版MQTT-SN，主要针对嵌入式设备

提供了一对多的消息分发机制。消息不是直接从发送器发送到接收器（即点对点），而是由MQTT server（即 MQTT Broker）分发的。

（来源：https://www.emqx.com/zh/blog/the-easiest-guide-to-getting-started-with-mqtt）

QoS

MQTT 提供了三种服务质量（QoS），在不同网络环境下保证消息的可靠性。

  • QoS 0：消息最多传送一次。如果当前客户端不可用，它将丢失这条消息。
  • QoS 1：消息至少传送一次。可以保证收到消息，但可能重复
  • QoS 2：消息只传送一次。可保证消息既不丢失也不重复

QoS 由 publisher 在 PUBLISH 报文中指定。大部分情况下 Broker 向订阅者转发消息时都会维持原始的 QoS 不变。不过也有一些例外的情况，根据订阅者的订阅要求，消息的 QoS 等级可能会在转发的时候发生降级。

例如，订阅者在订阅时要求 Broker 可以向其转发的消息的最大 QoS 等级为 QoS 1，那么后续所有 QoS 2 消息都会降级至 QoS 1 转发给此订阅者，而所有 QoS 0 和 QoS 1 消息则会保持原始的 QoS 等级转发。

（来源：https://www.emqx.com/zh/blog/introduction-to-mqtt-qos）

坐标变换

发表于 2024-11-27 本文字数： 732 阅读时长 ≈ 1 分钟

矢量控制时，三相同时工作；而方波控制时，同时只有两相工作。

控制 PMSM 也就是控制三相的磁场的强度和极性。三相绕组通电，定子磁场是三相电流产生磁场
的叠加。三个磁场耦合，分析比较困难。可通过坐标变换，把三相解耦，将交流量转换为直流量。

电机的磁链、电流、电压都是旋转的矢量，直接控制交流的矢量比较困难。将交流矢量转换 成幅值恒定 的直流量，可以简化控制。

PMSM 控制算法中采用的坐标变换通常包括静止坐标变换（Clark 变换）和同步旋转坐标变换（Park 变换）。

坐标系

坐标变换理论：如果三种坐标系中旋转磁场强度、速度相同，即磁动势相等，可认为等效，不影响机电能量转换。

在 dq 旋转坐标系内控制定子电流，从而方便地控制电磁转矩。所以需要坐标变换理论实现从三相静止坐标系向dq旋转坐标系的转化。

其中，ABC 为三相坐标系，α-β 是静止坐标系（α 与 A 相重合，β 与 α 垂直，逆时针旋转 90°），d-q 为同步旋转坐标系（d 轴为直轴，代表励磁分量，q 轴为交轴，代表力矩分量）。

定子磁势可分解为d轴磁势和q轴磁势，d轴磁势与转子磁势同轴，不能产生切向的力矩，但会影响永磁同步电机转子永磁体所产生的磁场；q轴与转子磁势相差90度，因而产生切向的力矩（类似两根垂直的条形磁铁所产生的相互作用力）。

Converter_Three-Phase

发表于 2024-11-21 本文字数： 1.2k 阅读时长 ≈ 2 分钟

Converter（Three-Phase）模块模型是由三个桥臂(three bridge arms)组成的六脉冲三相控制转换器。每个桥臂包括两个开关装置。转换电路将三相交流网络连接到直流网络。

  • G：Gate terminals。与开关器件的闸端相连接的矢量输入端口。将此端口连接到 Six-Pulse Gate Multiplexer 块
  • ~：Three-Phase port。复合三相端口，参考：Three-Phase Ports
  • a：a-phase
  • b：b-phase
  • c：c-phase
  • +：与直流正极相连的电保护端口
  • -：与直流负极相连的电保护端口
  • H：R2024b 新增，与开关器件的热质量有关的保温口

Switching device 选项：

  • GTO：门极可关断晶闸管。是一种具有自关断能力和晶闸管特性的晶闸管。 如果在阳极加正向电压时，门极加上正向触发电流，GTO就导通。 在导通的情况下，门极加上足够大的反向触发脉冲电流，GTO就由导通转为阻断

  • Ideal semiconductor switch：理想半导体开关

  • IGBT：绝缘栅双极晶体管（Insulated-gate bipolar transistor）。是一个非通即断的开关，是由 BJT（双极型三极管）和 MOS（绝缘栅型场效应管）组成的复合全控型电压驱动式功率半导体器件，兼有 MOSFET 的高输入阻抗和 GTR（Giant Transistor—GTR，巨型晶体管）的低导通压降两方面的优点。GTR饱和压降低，载流密度大，但驱动电流较大；MOSFET驱动功率很小，开关速度快，但导通压降大，载流密度小

  • MOSFET：绝缘栅型场效应晶体管（IGFET）。开关速度快，输入阻抗高，热稳定性好；但电流容量小，耐压低

  • Thyristor：晶闸管。有三个 PN 结。

    • 导通条件：（需同时满足）
      • 正向阳极电压
      • 门极有触发电流
    • 关断条件：（两种方法选择一种）
      • 阳极施加反向电压
      • 阳极电流降到接近于零的某一数值（维持电流）以下

  • Averaged Switch：带反并联二极管的半导体开关。控制信号端口 G 接受区间 [0,1] 中的值。当 G 等于 0 或 1 时，平均开关分别完全打开或完全关闭。该开关的行为类似于带有反并联二极管的 Ideal Semiconductor Switch 模块。当 G 介于 0 和 1 之间时，平均开关部分打开。您可以平均指定周期内的脉宽调制 （PWM） 信号。然后，您可以对模型进行欠采样，并使用调制波形而不是 PWM 信号
